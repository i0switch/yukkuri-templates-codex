# 02_演出編集プロンプト

## 目的

このプロンプトは、生成済みの台本をもとに、ゆっくり解説・ずんだもん解説動画の **編集演出指示** を作るためのものです。

このファイルでは、**どう見せるか・どう動かすか・どこに効果音を入れるか・字幕をどう表示するか** を扱います。
台本の内容そのものは原則として書き換えません。

---

## 重要な分離ルール

以下はこのファイルで扱います。

- 効果音
- BGM
- 字幕フォント
- 字幕縁取り
- 外枠・字幕帯
- 章切り替え演出
- メイン画像表示演出
- キャラのリアクション
- 図解・画像・表・グラフの出し方
- OP / ED の画面演出
- Remotion / YMM4 / AviUtl / DaVinci Resolve 等に落とし込める編集仕様

以下は原則として扱いません。

- 台本の新規作成
- 台詞の大幅な書き換え
- 章構成の根本変更
- 事実関係の追加説明

ただし、字幕として長すぎる台詞を短く分割する、または表示単位を整理することは許可します。

演出加工後に `script.yaml` を更新した場合は、動画生成へ進む前に必ずプリチェックを通します。

```powershell
node scripts/lint-script-pre.mjs <episode_id>
```

このチェックで `meta.layout_template`、`sub: null`、`title_text`、`assets/...` 相対パス、25文字制限を先に潰します。

---

## 入力情報

以下を受け取って、編集演出指示を作成してください。

```text
台本ファイル：{{SCRIPT_FILE}}
動画タイプ：ゆっくり解説 / ずんだもん解説
想定尺：{{DURATION}}
画面比率：16:9 / 9:16 / 1:1
解像度：1920x1080 / 1080x1920 など
編集環境：Remotion / YMM4 / AviUtl / DaVinci Resolve / 未定
利用可能素材：{{AVAILABLE_ASSETS}}
利用可能フォント：{{AVAILABLE_FONTS}}
利用可能SE：{{AVAILABLE_SE}}
利用可能BGM：{{AVAILABLE_BGM}}
```

未入力の場合は、以下を標準とします。

```text
画面比率：16:9
解像度：1920x1080
編集環境：Remotion想定
通常字幕フォント：けいふぉんと
通常字幕：白文字 + 太い黒縁 + 軽い影
```

# 1. 全体編集タイムライン

## 標準構成

| パート | 時間目安 | 編集目的 | 基本演出 |
|---|---:|---|---|
| OP | 5〜12秒 | 興味付け | タイトル表示、メイン画像チラ見せ、短いSE |
| 導入挨拶 | 10〜25秒 | 空気作り | キャラ登場、軽いBGM、通常字幕 |
| 本題提示 | 15〜30秒 | テーマ認識 | メイン画像ドン、タイトル補助 |
| お品書き | 5〜10秒 | 全体像提示 | 章立てカード |
| 本編 | 章ごと | 解説 | 画像、図解、強調字幕、キャラ反応 |
| まとめ | 30〜60秒 | 要点回収 | 箇条書き、結論テロップ |
| ED | 10〜25秒 | CTA | 登録誘導、次回予告、フェードアウト |

# 2. OP編集ルール

## 目的

最初の数秒で「この動画は見てもよさそう」と思わせる。

## 推奨演出

```text
黒背景 or 背景ぼかし
↓
メインテーマ画像を一瞬表示
↓
タイトルを大きく表示
↓
霊夢・魔理沙、またはずんだもん・めたんを表示
↓
本編へ移行
```

## パラメータ目安

| 項目 | 推奨値 |
|---|---|
| OP尺 | 5〜12秒 |
| タイトルズーム | 92% → 100% |
| フラッシュ | 0.10〜0.15秒。使いすぎ禁止 |
| シェイク | 1回まで |
| SE | ドン / シャキーン / ジャキーン |
| BGM | OP専用。短く派手め |

---

# 3. 章切り替え演出

章切り替えは、視聴者の集中力をリセットするために必ず入れます。

## 基本構造

```text
前章のラスト台詞
↓
短い無音またはBGMフェード
↓
章タイトルカード 1.0〜1.8秒
↓
SE
↓
次章の最初の台詞
```

## 章タイトルカード

| 要素 | 指定 |
|---|---|
| 表示時間 | 1.0〜1.8秒 |
| 背景 | 暗め背景 / 半透明帯 / ぼかし背景 |
| 章番号 | 大きく表示 |
| タイトル | 中央表示 |
| SE | ポン / カキン / パキーン / シャキーン |
| BGM切替 | 雰囲気が変わる章のみ |

## 注意

毎回派手なフラッシュやシェイクを入れると安っぽくなるため、章ごとに強弱をつけてください。

# 4. メイン画像表示演出

## 初登場時

| 項目 | 指定 |
|---|---|
| 登場 | ズームアウト 120% → 100% |
| フェード | 0.2〜0.3秒 |
| 枠 | 黒枠 2〜4px |
| 影 | 軽いドロップシャドウ |
| SE | キラン / シャキーン / ドン |
| 表示時間 | 3〜6秒 |

## 通常表示時

| 用途 | 演出 |
|---|---|
| 説明画像 | フェードイン |
| 背景資料 | ゆっくりパン・ズーム |
| 比較画像 | 左右スライドイン |
| 重要画像 | 赤枠または黄色ハイライト |
| 不穏な資料 | 背景暗め、彩度低め |
| 古い資料 | セピア、紙質感、軽いノイズ |

# 5. 字幕・フォント設計

## 基本方針

ゆっくり解説の通常字幕は、以下を標準とします。

```text
フォント：けいふぉんと
文字色：白
外縁：黒
外縁太さ：太め
影：軽め
位置：画面下中央
行数：最大2行
```

---

## 通常字幕

### フルHD 1920x1080 目安

| 項目 | 目安 |
|---|---|
| フォント | けいふぉんと |
| 文字サイズ | 60〜76px |
| 黒縁 | 6〜10px |
| 影 | うっすら |
| 行数 | 最大2行 |
| 配置 | 下部中央 |
| 下余白 | 画面下端から少し上げる |

---

## 外枠囲い字幕

背景が複雑な場合は、字幕の背面に外枠または帯を追加します。

### パターンA：極太縁取り型

```text
白文字
太い黒縁
軽い影
背景なし
```

向いている場面：

- 背景がシンプル
- 昔ながらのゆっくり解説感を出したい
- 字幕を軽く見せたい

### パターンB：字幕帯型

```text
半透明の黒帯
白文字
黒縁
```

向いている場面：

- 背景がごちゃつく
- 画像素材が派手
- 説明をしっかり読ませたい

### パターンC：角丸ボックス型

```text
角丸の半透明ボックス
白文字
黒縁
話者タグ付き
```

向いている場面：

- ずんだもん系
- 今っぽい解説動画
- 字幕をデザインとして統一したい

---

## 話者タグ

本文色をキャラごとに変えるより、話者タグや帯色で区別します。

| キャラ | タグ色 | 本文 |
|---|---|---|
| 霊夢 | 赤系 | 白文字 + 黒縁 |
| 魔理沙 | 黄系 | 白文字 + 黒縁 |
| ずんだもん | 緑系 | 白文字 + 黒縁 |
| 四国めたん | 紫・ピンク系 | 白文字 + 黒縁 |

本文そのものを赤や黄色にしすぎると画面がうるさくなるため、基本は白文字で統一します。

---

## 強調字幕

| 種類 | 表示 |
|---|---|
| 重要ワード | 黄色文字 + 黒縁 + 軽い拡大 |
| 危険・NG | 赤文字 + 黒縁 + 背景暗転 |
| 結論 | 中央に大きく表示 + ドンSE |
| ツッコミ | 斜め配置 + 大きめ文字 + ビシッSE |
| 数字 | カウントアップ + 太字 |

---

## 改行ルール

```text
1字幕 = 1意味
最大2行
1行は短く
変な位置で改行しない
読みにくい長文は分割する
```

# 8. BGM設計

## 必要なBGM

| パート | BGM |
|---|---|
| OP | 短く派手 |
| 導入 | 明るめ・軽め |
| 通常解説 | 邪魔しないループ |
| 不穏・深掘り | 低音・緊張感 |
| まとめ | 落ち着いた曲 |
| ED | 明るい締め |

## 音量方針

- セリフを最優先にする
- BGMはセリフを邪魔しない
- SEは瞬間的に目立つ程度
- BGM切替時は0.3〜1.0秒程度でフェードする
- クライマックス前に一瞬無音を作ると、次の一言が刺さりやすい

# 11. 品質チェック

出力後、以下を自己チェックしてください。

- 台本の内容を書き換えすぎていないか
- 各シーンに最低1つは視覚的な役割があるか
- 章切り替え演出が入っているか
- メイン画像初登場時の演出があるか
- 字幕フォントと縁取りルールが統一されているか
- けいふぉんと + 白文字 + 黒縁を基本にしているか
- 背景が派手な場面で字幕帯や外枠が使われているか
- SEが多すぎないか
- BGMがセリフを邪魔しない設計か
- 素材・フォント・BGM・SEのライセンス確認メモがあるか
