# 08_image_prompt_prompt

## 目的

`script_final.md` の対象シーン全文をそのまま使い、ChatGPT Images 2.0へ渡す画像生成プロンプトを作る。

画像生成プロンプト本文には、対象シーン全文、固定プロンプト、台本に合わせた雰囲気指定以外を入れない。

出力先は `image_prompt_v2.md` を基本にする。
長文プロンプトが `script.yaml` を圧迫する場合は、`image_prompts.json` に逃がしてよい。
その場合、`script.yaml` の `visual_asset_plan[]` には `imagegen_prompt_ref` を置く。
既存互換として、`visual_asset_plan[].imagegen_prompt` に直接書く形式も使える。

## 必須入力

- episode全体のテーマ
- `script_final.md` 全体
- 対象シーンID
- 対象シーンタイトル
- 対象シーンの会話全文
- 台本全体から判断した画像の雰囲気

## 必須項目

- 対象シーンIDとタイトル。
- 対象シーンの会話全文。
- 対象シーンタイトルを画像内見出しとして必ず大きく目立たせる明示指示。scene id は画像内に入れない。
- 固定プロンプト。
- LLMが台本全体から判断した雰囲気指定。
- 画像の役割: 理解補助 / 不安喚起 / 笑い / 比較 / 手順整理 / 証拠提示 / オチ補助 のいずれか。
- 構図タイプ: NG / OK 比較 / 失敗例シミュレーション / 誇張図解 / 証拠写真風 / チェックリスト / 手順図 / 原因マップ / ビフォーアフター / ツッコミ待ち構図 / 事故寸前構図 のいずれか。
- 画面で一番大きく見せる対象、その理由、小さくする/入れない対象。
- 画像化する一言、画像で再現する状況、画像で再現しない要素。

`image_role` と `composition_type` は `visual_asset_plan[]` の公開メタとして必須。
ただし `imagegen_prompt` 本文には `image_role:` や `composition_type:` のような英語キーは入れない。
プロンプト本文では、日本語の自然文として「画像の役割」「構図タイプ」を指定する。

## 完成形式

```yaml
visual_asset_plan:
  - slot: main
    purpose: "script_final直投げ型の挿入画像"
    adoption_reason: "対象シーンの会話全文から、シーンの状況と感情を直接画像化するため"
    image_role: "理解補助"
    composition_type: "誇張図解"
    imagegen_prompt_ref: "image_prompts.json#s01.main"
    imagegen_prompt: |-
      {scene_id}: {scene_title}

      {script_final.mdから対象シーン全文をそのまま貼る}

      画像内に「{scene_title}」という見出しを必ず目立つように配置してください。

      ゆっくり解説動画向けの挿入画像を日本語で生成してください。 この画像は会話内容をそのまま再現するためのものではなく、シーンの要点・状況・概念・比喩を視覚的にわかりやすく補強するためのコンテンツ画像です。 字幕やセリフは別で表示するため、会話等は画像に入れないでください。 キャラクター同士の会話シーンにはせず、テーマ理解を助ける図解、アイコン、小物、抽象的な画面風ビジュアル、概念図、状況説明ビジュアルを中心に構成してください。 画像内の可読テキストは日本語だけにしてください。英語ラベル、英語見出し、英語UI、英単語の装飾文字は禁止です。 画像内には、対象シーンの見出し本文だけを大きく目立つ見出しとして配置してください。scene idは画像内に入れないでください。 文字が崩れる可能性がある場合は、文字を使わずアイコン、色分け、形、配置で表現してください。 下部に白帯、入力欄、チャット欄、テキストボックス風の余白を作らないでください。 画像の役割を、理解補助、不安喚起、笑い、比較、手順整理、証拠提示、オチ補助のいずれかとして明確にしてください。 構図タイプを、NG / OK 比較、失敗例シミュレーション、誇張図解、証拠写真風、チェックリスト、手順図、原因マップ、ビフォーアフター、ツッコミ待ち構図、事故寸前構図のいずれかとして明確にしてください。 画面で一番大きく見せる対象、その対象を大きくする理由、逆に小さくする/入れない対象を明確にしてください。 対象シーンから画像化する一言を1つ選び、画像で再現する状況と再現しない要素を明確にしてください。 スマホ視聴でも伝わるように主役は1つ、必須見出し以外の画像内の小さい文字は最大3語、アイコンは最大5個、重要要素は画面中央から上部に置いてください。 どのシーンにも使える白背景アイコン、抽象的な青いネットワーク線、台本の具体例と関係が薄い綺麗な汎用図解は禁止です。 字幕やキャラクターに重なる位置へ重要情報を置かず、背景や小物は画面端まで自然に続けてください。 画面全体を有効活用し、情報が一目で伝わる、整理された高品質なビジュアルにしてください。 動画の解説画面に適した、見やすく印象的で、内容理解を助ける16:9の横長構図で作成してください。

      画像の雰囲気は{LLMがscript_final.md全体から判断した雰囲気}で生成してください。
```

`imagegen_prompt_ref` と inline `imagegen_prompt` は同じ内容を指す。
長文を `image_prompts.json` に分離する場合は `imagegen_prompt_ref` を使い、inline `imagegen_prompt` は省略してよい。
inline 形式を使う場合でも、内容は `image_prompt_v2.md` または `image_prompts.json` と同じ直投げ型プロンプトにする。

`image_prompts.json` を使う場合の最小形:

```json
{
  "version": 1,
  "prompts": [
    {
      "scene_id": "s01",
      "slot": "main",
      "imagegen_prompt": "{image_prompt_v2.md と同じプロンプト}"
    }
  ]
}
```

## 方針

新規生成では `image_direction`、`visual_type`、`supports_dialogue`、`supports_moment`、`hook_type`、`myth_vs_fact`、`boke_or_reaction` は使わない。
これらは旧抽象タグであり、`visual_asset_plan[]` の必須項目にも、`imagegen_prompt` 本文にも入れない。

sub 用 imagegen_prompt は作らない。
sub枠ありテンプレートでは、sub は YAML 側の `sub.kind: text` または `sub.kind: bullets` で扱う。
main は画像、sub は補助テキスト/箇条書きとして責任を分ける。
旧固定文に含まれていた英語のアスペクト比指定、下部余白指定、UI単独指定は使わない。

## 禁止

- 会話全文を画像内に並べる指示。
- scene id を画像内見出しに含める指示。
- 字幕用の白帯、入力欄、チャット欄、テキストボックス風余白を作る指示。
- どのシーンにも使える白背景アイコン、抽象的なネットワーク線、汎用フラット図解。
- 実在UI、実在ブランドロゴ、実在人物の模写。
- `image_direction`、`visual_type`、`supports_dialogue`、`supports_moment`、`hook_type`、`myth_vs_fact`、`boke_or_reaction` を新規プロンプトへ混ぜること。
