# example_bad_vs_good_image_prompt

画像プロンプト（`imagegen_prompt`）の anti-pattern と、その書き直し例。

参照: `00_MASTER_SCRIPT_RULES.md` の「画像プロンプト」「禁止」。

`imagegen_prompt` には `script_final.md` の対象シーン全文を主入力として入れる。1プロセス=1画像、16:9、コンテンツ挿入用の素材。会話字幕は別表示なので画像内に会話を並べない。

---

## anti-pattern 1: 画像プロンプトに `visual_type: hook_poster` などの抽象タグが混入

テーマ: スマホの写真整理 Scene01

### Bad（不採用）

```
visual_type: hook_poster
hook_type: loss_aversion
supports_dialogue: true
supports_moment: opening_shock
composition_type: split_screen
boke_or_reaction: reiimu_panic
image_direction: warm_modern_flat

スマホ画面に2万枚と表示されている画像を作って
```

### なぜダメか

- v2 では使用しない抽象タグ（`visual_type` `hook_type` `supports_dialogue` `composition_type` 等）が混入（必ずFAIL: 画像プロンプトが抽象的）
- 画像生成モデル（gpt-image-1）はこれらタグを正しく解釈できず、無視するか誤解する
- 「スマホ画面に2万枚」だけが本体で、シーン全文の文脈が無い
- 結果として汎用ストック画像のような出力になる

### Good（採用）

```
以下のシーン全文の雰囲気とテーマに合わせ、16:9のコンテンツ挿入画像を作る。
日本語テキストは画像内に入れてよいが、会話全文を並べない。
作風はシーン全文から判断し、温かみのある現代的なトーンで描く。

【シーン全文】
霊夢「魔理沙、写真消そうとしたら『2万枚』って出たんだけど」
魔理沙「2万は危険水域だぜ。そのまま全選択削除すると、消したくないやつまで飛ぶ」
霊夢「えっ、もう全選択ボタン押す寸前だったんだけど！？」
魔理沙「待て待て、消す前に見る場所が3つある。そこを通せば事故らない」

【描写指示】
スマホ画面のクローズアップ。ギャラリーアプリの上部に「20,194枚」と日本語で大きく表示。
画面右下に「全選択」「削除」ボタンが赤く強調されて見える構図。
机の上、自然光、手前にぼかしで湯呑みかメモ帳を入れて生活感を出す。
警告の緊張感が伝わるように、画面に薄い赤いオーバーレイを入れる。
```

### Good の何が違うか

- v2 で禁止されている抽象タグ（visual_type 等）がゼロ
- シーン全文をそのまま入れて、画像モデルに文脈を渡している
- 「20,194枚」と日本語の具体テキストで一目で意図が伝わる
- 「全選択ボタンが赤く強調」「警告の緊張感」と画面内の意味が指定されている
- 16:9 / 会話全文を並べない / 字幕は別表示、という v2 ルールに準拠

## anti-pattern 2: 「白背景中央にアイコン、シンプル図解」など汎用指定

テーマ: 冷凍ご飯のラップ巻き方 Scene03

### Bad（不採用）

```
白背景の中央にご飯のアイコンを配置。
シンプルなフラットイラスト調で、アイコン下に「冷凍ご飯」とテキスト。
余白は広めに、ミニマルなインフォグラフィック風で。
```

### なぜダメか

- 「白背景中央アイコン」「汎用フラット図解」（必ずFAIL: 画像プロンプト禁止リスト）
- ストック素材のアイコン1個と区別がつかない
- シーンの内容（ラップの巻き方）が画像から読み取れない
- ゆっくり/ずんだもん動画のメイン画像として安っぽく見える

### Good（採用）

```
以下のシーン全文の雰囲気とテーマに合わせ、16:9のコンテンツ挿入画像を作る。
日本語テキストは画像内に入れてよいが、会話全文を並べない。
作風はシーン全文から判断し、温かい家庭料理感のあるトーンで描く。

【シーン全文】
ずんだもん「ラップ……ずんだもん、適当にふんわり包んでたのだ」
めたん「それが石の正体ね。空気が入ると、そこから水分が抜けて固くなる」
ずんだもん「じゃあピタッと巻けばいいってこと？」
めたん「そう。1cm厚に平たくして、空気抜いてピッチリ。ここで仕上がりが決まる」

【描写指示】
木のまな板の上、炊きたての白ご飯をラップで包む手元のクローズアップ。
左右に2つの状態を並べ、左は「ふんわり包んで空気入り（NG）」、右は「平たく1cm厚で空気抜き（OK）」。
それぞれの上に「NG」「OK」と日本語で表示。湯気が立っている描写を加え、
温かさが画面から伝わるようにする。背景は柔らかい木目で生活感を出す。
```

### Good の何が違うか

- 白背景・中央アイコン・汎用フラット図解の禁止に該当しない
- シーン全文を主入力に渡し、文脈から作風が決まるようにしている
- NG/OK の比較構図でシーン内容を視覚化
- 「1cm厚」「空気抜き」など具体ポイントが画像で伝わる
- 「湯気」「木目」「生活感」と質感まで指定し、量産アイコン感を回避

## anti-pattern 3: 1つの画像プロンプトに「8枚一括生成」「グリッド」など batch 指示

テーマ: 写真整理 5分動画の全シーン素材

### Bad（不採用）

```
以下8シーン分の画像をまとめて生成してください。
2x4のグリッドにレイアウトし、1枚のsprite sheetとして出力。
各セルに以下の内容を割り当てる:
- セル1: スマホに2万枚警告
- セル2: 重複写真の例
- セル3: スクショ画面
- セル4: 動画ファイル一覧
- セル5: クラウドアップロード
- セル6: 自動整理アプリ画面
- セル7: 月1ルーティンのカレンダー
- セル8: 明日の朝やる1個のメモ

それぞれを正方形でcrop可能にして、後でasset sheetから切り出して使います。
```

### なぜダメか

- 「8枚一括生成」「グリッド」「sprite sheet」「asset sheet」「crop / 切り出し」が全部禁止項目（必ずFAIL）
- 「1つの codex exec プロセスに複数画像のプロンプトをまとめて渡す」も禁止（1プロセス=1画像厳守）
- gpt-image-1 で grid を出力させると、各セルの解像度が下がり、文字が潰れる
- 後で切り出す前提では、16:9 のメイン画像として使えない
- 1枚あたりの密度・整合性が保てず、全シーンの素材が低品質になる

### Good（採用）

各シーンごとに独立した画像プロンプトを作る。1ファイル=1シーン=1プロセス=1画像。

例として Scene01 の場合:

```
以下のシーン全文の雰囲気とテーマに合わせ、16:9のコンテンツ挿入画像を1枚作る。
日本語テキストは画像内に入れてよいが、会話全文を並べない。
グリッド・複数枚・sprite sheet にはしない。1枚で完結する画像にする。
作風はシーン全文から判断し、温かみのある現代的なトーンで描く。

【シーン全文】
霊夢「魔理沙、写真消そうとしたら『2万枚』って出たんだけど」
魔理沙「2万は危険水域だぜ。そのまま全選択削除すると、消したくないやつまで飛ぶ」
霊夢「えっ、もう全選択ボタン押す寸前だったんだけど！？」
魔理沙「待て待て、消す前に見る場所が3つある。そこを通せば事故らない」

【描写指示】
スマホ画面のクローズアップ。ギャラリーアプリ上部に「20,194枚」と日本語で大きく表示。
右下に「全選択」「削除」ボタンが赤く強調された構図。
机の上、自然光、薄い赤の警告オーバーレイ。
```

Scene02 以降も同様に、それぞれ独立した imagegen_prompt を持ち、それぞれ独立した codex exec プロセスで生成する。

### Good の何が違うか

- 1プロセス=1画像、グリッド・sheet・crop 一切なしで v2 ルールに準拠
- 各シーンの imagegen_prompt が独立しており、1枚あたりの解像度と密度が保てる
- 16:9 で完結するため、後工程（Remotion 配置）でそのまま使える
- 並列で生成したい場合は、シーンごとに別プロセスを起動する運用にする
- 失敗時も該当1枚だけ再生成すればよく、再生成コストが小さい
