# example_bad_vs_good_structure

構成・密度の anti-pattern と、その書き直し例。

draft / rewrite 工程では、章タイトル・冒頭・中盤・結末がここに引っかかっていないかを必ず照合する。

参照: `00_MASTER_SCRIPT_RULES.md` の「FAIL条件」「章タイトル」「基本方針」。

---

## anti-pattern 1: 章タイトルが「〇〇とは / 〇〇について」（フック型でない）

テーマ: スマホのバックアップ

### Bad（不採用）

```
Scene01: バックアップとは
Scene02: バックアップの種類について
Scene03: クラウドバックアップについて
Scene04: ローカルバックアップとは
Scene05: バックアップの頻度について
Scene06: まとめ
```

### なぜダメか

- すべての章タイトルが「とは / について」型（必ずFAIL: 章タイトルが説明目次）
- 視聴者が「次の章で何が起きるのか」を予測できない
- 教科書の目次であり、コンテンツの引きが消えている
- サムネ・チャプター送りからの流入も逃す

### Good（採用）

```
Scene01: バックアップ済みの罠
Scene02: 写真だけ守って詰むパターン
Scene03: クラウドが守ってくれない3種類
Scene04: 機種変前夜にやる5分の儀式
Scene05: 容量0でも逃がす裏ルート
Scene06: 明日からの最小セット
```

### Good の何が違うか

- 「罠」「詰むパターン」「儀式」「裏ルート」と動詞・損失語が入っている
- 各章で「何が起きるか」が予測できる
- Scene02 の「写真だけ守って詰むパターン」で、安心していた人ほど刺さる
- 章タイトル単体でもクリックされる強度

## anti-pattern 2: 「今回は〇〇について解説します」で始まる

テーマ: 冷凍ご飯（ZMペア想定）

### Bad（不採用）

```
Scene01 オープニング:
めたん「今回は冷凍ご飯のおいしい作り方について解説するわ」
ずんだもん「お願いするのだ」
めたん「冷凍ご飯は正しく作れば、炊きたてに近い食感が戻るのよ」
ずんだもん「楽しみなのだ」
```

### なぜダメか

- 「今回は〇〇について解説します」開始（必ずFAIL）
- 冒頭5秒に損失・意外性・誤解訂正・強い疑問・数字なし
- ずんだもんが「お願いするのだ / 楽しみなのだ」のみ（FAIL: ZMリアクション単調）
- 視聴維持率が冒頭で崩れ、以降の内容も見られない

### Good（採用）

```
Scene01 オープニング:
ずんだもん「冷凍ご飯、レンチンしたらカチカチで石みたいになったのだ」
めたん「あぁ、それ作り方の問題ね。冷凍前の3秒で全部決まるわよ」
ずんだもん「3秒！？じゃあ今までの石、ずんだもんのせいじゃないってこと！？」
めたん「8割がたそう。ラップの巻き方と、温度落ちる前に冷凍庫入れるかどうか、ここが分かれ目」
ずんだもん「巻き方……ずんだもん、適当にふんわり包んでたのだ」
めたん「それが石の正体ね」
```

### Good の何が違うか

- 「カチカチで石」と具体失敗で開始（損失提示）
- 「3秒で全部決まる」という意外性ある数字フック
- ずんだもんの「ずんだもんのせいじゃない！？」が当事者リアクション
- 「ラップの巻き方」「温度落ちる前」と具体ポイント2つ
- 解説宣言なしで本編に直接入っている

## anti-pattern 3: 中盤再フックなし、平坦に進む

テーマ: 写真整理（5分動画想定、Scene数10）

### Bad（不採用）

```
Scene01: 写真2万枚で困る話
Scene02: まず内訳を見る
Scene03: 重複写真を消す
Scene04: スクショを消す
Scene05: 動画を整理する  ← 中盤、新フックなし
Scene06: クラウドに上げる ← 中盤、新フックなし
Scene07: アプリを使う
Scene08: 自動整理を設定
Scene09: 月1で見直す
Scene10: まとめ
```

### なぜダメか

- 中盤40〜60%地点に再フックなし（FAIL: 中盤再フックなし）
- Scene05〜Scene06 が「次のテクニック」を並べているだけ
- 視聴者が「もう内容把握した」と感じて離脱する地点
- 後半の Scene07以降の良い情報まで届かない

### Good（採用）

```
Scene01: 写真2万枚で容量警告
Scene02: 消す前に見る3つの場所
Scene03: 重複の落とし穴
Scene04: スクショ捨ての境界線
Scene05: ここまでで7000枚減ったが、本当の犯人は別  ← 中盤再フック
Scene06: 動画1本=写真300枚問題  ← 数字で再フック
Scene07: クラウド任せが危険な理由
Scene08: 自動整理の落とし穴
Scene09: 月1ルーティン3分
Scene10: 明日の朝やる1個だけ
```

### Good の何が違うか

- Scene05 で「本当の犯人は別」と再フック、視聴を引き戻す
- Scene06 「動画1本=写真300枚」と意外性ある比較
- 中盤地点で「まだ続きを見る価値がある」と示せている
- 章タイトルがすべてフック型
- Scene10 が「明日の朝やる1個」と具体行動に落ちている

## anti-pattern 4: 最後が「以上です」「ありがとうございました」で行動指示なし

テーマ: 食費節約

### Bad（不採用）

```
Scene最終:
魔理沙「というわけで、食費を抑える方法は3つあるんだぜ」
霊夢「勉強になったよ」
魔理沙「ぜひ試してみてくれ」
霊夢「以上、本日はありがとうございました」
魔理沙「またな」
```

### なぜダメか

- 「以上です / ありがとうございました」で締める（FAIL: 具体行動で終わらない）
- 「ぜひ試してみてくれ」が抽象的で、視聴者は何もしない
- 「3つあるんだぜ」と数だけ言って中身を再強調していない
- 視聴後の行動転換率がゼロに近い

### Good（採用）

```
Scene最終:
魔理沙「というわけで、明日からやることを1個だけに絞るぜ」
霊夢「1個でいいの？3個全部やろうとしてた」
魔理沙「3個やろうとして全部失敗するのが一番もったいないんだぜ。明日の買い物前に5分だけ、リスト作る。これだけ」
霊夢「リスト1枚なら今夜書ける。冷蔵庫の写真撮ってからスーパー行く」
魔理沙「それだぜ。1週間後、レシート見比べてみてくれ。たぶん2000円は違う」
霊夢「2000円違ったら、来月のおやつ予算が増える」
```

### Good の何が違うか

- 「明日の買い物前に5分」と時刻と所要時間が具体
- 「冷蔵庫の写真撮ってからスーパー」と動作レベルの行動
- 「1週間後、レシート見比べる」と検証アクション付き
- 「2000円違う」と期待値が数字で見える
- 「以上です」「ありがとうございました」を使わずに自然に締めている
- 霊夢の最後の一言が動機づけになっている
