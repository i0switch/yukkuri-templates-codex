# 01_plan_prompt

## 目的

台本を書かずに、企画、構成、テンプレート枠の使い方、掛け合いの設計だけを決める。
必ず `00_MASTER_SCRIPT_RULES.md` を先に読む。

## 入力

```text
テーマ：
ざっくり入れたい内容：
想定尺：
使用テンプレート：
キャラペア：RM / ZM
想定視聴者：
避けたい内容：
参考資料：
```

## やること

1. 不足情報と仮定を整理する。
2. 選択テンプレートの main / sub / subtitle / title / avoid_area を要約する。
3. 採用する企画角度を1つ決める。
4. 掛け合い方針を決める。
5. 視聴者代表キャラの主な勘違いを決める。
6. ボケパターンを3案出す。
7. 冒頭フックの方向性を3案出す。
8. 章タイトルのフック型を決める。
9. 各シーンの `scene_format` を決める。
10. 各シーンに入れる数字・具体例・あるあるを決める。
11. どのシーンでL3以上のリアクションを入れるか決める。
12. どのシーンで例え話・小ネタを入れるか決める。
13. 想定尺に対する本編シーン数、目標セリフ数、1シーン平均セリフ数を決める。
14. 各章の `scene_goal`、`viewer_question`、`visual_role` を決める。
15. 各章で必要な main / sub 素材の方針を決める。
16. 各章の `imagegen_prompt` で使う主題、構図、色、余白、禁止要素を決める。
17. episode共通の `meta.image_style` を決める。
18. 参照動画型なら `reference_style` と主要 `reference_beat` を決める。
19. 最終シーンで、視聴者に実行させる具体行動を2アクション以上で設計する。
20. 章タイトルがテーマ語彙から浮いていないか確認する。
21. キャラ口癖の想定比率を決める。
22. 自然文チェックで見落としやすい表現を最後に確認する前提を置く。

## 設計ルール

- 想定尺に合わせて、最初に本編シーン数と本編セリフ数の目標を決める。
- 5分前後なら10〜12シーン、90〜130セリフ、1シーン平均8〜12セリフを基準にする。
- 5分前後で8シーン以下、70セリフ未満の計画は作らない。
- 各シーンに最低1回、追加の深掘り往復を設計する。
- ZMでは、ずんだもんの「のだ」「なのだ」を20〜40%目安で使う。0%にしない。
- 章タイトルは、損失型、誤解訂正型、行動型、あるある型、比較型のいずれかにする。
- `scene_format` は動画全体で最低3種類以上使う。
- 視聴者代表キャラは、質問係ではなく、勘違い、反論、ボケ、言い換え、焦りを担当する。
- 解説役は2セリフ以内で一度、視聴者代表キャラへ返す前提で構成する。
- L3以上のリアクションは動画全体で最低2回配置する。
- 3から5シーンに1回、短い例え話・小ネタを入れる。
- 数字は根拠が曖昧なら「目安」「例」「場合がある」として扱う。
- 実在ブランドUIの模写を避け、必要に応じて「メッセージアプリ風」などに一般化する。
- 画像素材方針では、主役オブジェクト、補助オブジェクト、背景色、アクセント色、字幕帯/キャラ回避の余白を決める。
- `imagegen_prompt方針` は「フラットな図解」だけで済ませず、8項目フォーマットへ展開できる粒度にする。
- main枠は中央主題、sub枠は3項目以内の補足として役割を分ける。
- 画像内文字は原則なし。必要な場合も日本語3〜6文字までにする。
- 最終シーンは「見るだけ」で終わらせず、確認、1つ選ぶ、保存/逃がす/テンプレ化、予定化などから2アクション以上を設計する。
- 章タイトルはフック型であるだけでなく、テーマ外の単語が混ざっていないか確認する。
- ZMではずんだもんの「のだ」「なのだ」20〜40%、RMでは魔理沙の「だぜ」30〜60%を目安にする。
- 企画表の段階で、最終行動 `final_action` と、テーマ外タイトルチェック `title_theme_fit` を持たせる。
