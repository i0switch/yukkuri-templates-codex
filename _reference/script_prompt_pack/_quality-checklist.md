# 台本クオリティ・チェックリスト（共通）

`_reference/script_prompt_pack/{ゆっくり解説_霊夢魔理沙_台本生成プロンプト_改善版.md, ずんだもん解説_ずんだもんめたん_台本生成プロンプト_改善版.md}` の `# 8. 出力前セルフチェック` から参照される共通チェックリスト。
監査プロンプト（`台本監査_改善プロンプト.md`）からも `### 6. 意味的チェック` で同じ観点を使う。

## 0. 使い方

1. 台本生成完了前に、各項目を **PASS / FAIL** で判定する
2. 形式的な存在確認ではなく、**実機で機能しているか**で判定する
3. FAIL が 1 つでもあれば該当箇所を書き直し、再度この checklist を通す
4. 全項目 PASS したものだけを最終出力とする

「100 点満点の採点」ではなく **二値判定**であることに注意。甘い採点で通過させない。

---

## 1. 構造（必達）

| ID | チェック内容 | 判定 |
|---|---|---|
| S-01 | 冒頭5秒：1文目が「実は」「先に結論」「なんで」「知らないと損」型で始まっている | □ |
| S-02 | 冒頭5秒禁止句が混入していない（「今回は◯◯について解説します」「こんにちは」「ゆっくりしていってね」から始まっていない） | □ |
| S-03 | 中盤フック（台本の 40〜60% 地点）：「ここまで聞くと〜だけど、実は〜」型または「でも本当にヤバいのは〜」型の転換セリフが存在する | □ |
| S-04 | 終盤回答：「結局どうすればいいか」「結局どう見ればいいか」への明確な答えセリフが存在する | □ |
| S-05 | 視聴後一言：締めのセリフが「視聴後に誰かへ話したくなる」一文になっている | □ |

## 2. 会話の質（必達）

| ID | チェック内容 | 判定 |
|---|---|---|
| Q-01 | 「短い疑問 → 短い説明」だけのサイクルが 3 章以上連続していない | □ |
| Q-02 | 各章に「疑問 → 説明 → 反論 or 補足 → 新発見」の 4 層が最低 1 回ある | □ |
| Q-03 | 各章に「実は」「意外にも」「ところが」型の意外性ファクトが最低 1 つある | □ |
| Q-04 | 疑問役（霊夢 / ずんだもん）が章ごとに最低 1 回、脱線ボケ・自己ツッコミ・本音漏れのいずれかをしている | □ |
| Q-05 | 解説役（魔理沙 / めたん）が 3 セリフ以上連続で話していない | □ |
| Q-06 | 同じ内容の繰り返しが 3 回以上ない（リスト型のお題ヘッダ反復は除く） | □ |

## 3. 会話比率（客観測定）

`R`：疑問役のセリフ数、`M`：解説役のセリフ数、`Rc`：疑問役の総字数、`Mc`：解説役の総字数

| ID | チェック内容 | 目標 | 判定 |
|---|---|---|---|
| R-01 | セリフ数比 `M / R` | 1.3〜1.7 | □ |
| R-02 | 字数比 `Mc / Rc` | 1.3〜1.7 | □ |
| R-03 | 全 `dialogue[].text` の文字数 | 各 25 字以内 | □ |
| R-04 | 平均セリフ字数 | 12〜20 字 | □ |

## 6. 動画化前提（script.yaml レンダー整合）

| ID | チェック内容 | 判定 |
|---|---|---|
| V-01 | `meta.layout_template` が `Scene01`〜`Scene21` 形式（`scene-XX` 形式は不可）で、`scenes[].scene_template` が存在しない | □ |
| V-02 | サブ枠なしテンプレで `sub: null` になっている | □ |
| V-03 | サブ枠ありテンプレで `bullets` が 3 項目以内、画像は単純図のみ | □ |
| V-04 | asset パスが `assets/...` の相対形式（`script/{episode_id}/assets/` ではない） | □ |
| V-05 | `title_text` がタイトル枠付きテンプレ（`Scene04 / 08 / 12 / 15 / 16 / 17 / 19`）でのみ使われている | □ |
| V-06 | サブ枠なしテンプレで `sub` を立てていない（テンプレから判断） | □ |

---

## 7. 出力時の判定ルール

| 結果 | 挙動 |
|---|---|
| 全 PASS | そのまま出力 |
| 1 つでも FAIL | 該当箇所のみ修正版を作る（プロンプト本体の `## 11. 70 点未満なら自動改善` を流用） |
| 修正後に再チェック | この checklist を再度通す |
| 3 周しても FAIL が残る | ユーザーに「どの項目が達成困難か」を報告して相談する |

> 重要：100 点満点採点（プロンプト本体 `## 10. 自己監査`）はあくまで補助情報。
> 合格判定の主軸は本 checklist の二値判定。
