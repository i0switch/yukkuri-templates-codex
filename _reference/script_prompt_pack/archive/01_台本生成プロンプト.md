# 01_台本生成プロンプト

## 目的

このプロンプトは、ゆっくり解説・ずんだもん解説動画の「台本」を生成するためのものです。

このファイルでは、**何を話すか・どんな順番で説明するか・会話として面白いか**だけを扱います。
編集演出、効果音、BGM、画面エフェクト、Remotion/YMM4上の動きは指定しません。
ただし、動画化用の `script.yaml` を作る場合は、台本内容に合うフォント分類だけ `typography` で指定します。

---

## 重要な分離ルール

以下はこのファイルでは扱わないでください。

- 効果音の指定
- BGMの指定
- 字幕の装飾指定
- テロップの縁取り指定
- 画像のズーム、フェード、シェイクなどの映像演出
- 章切り替えアニメーション
- キャラのバウンス、口パク、目パチなどの編集指定
- Remotion / YMM4 / AviUtl / DaVinci Resolve の実装指定

このファイルで出すのは、あくまで **台本・構成・会話・素材候補の内容メモ** までです。

---

## 入力情報

以下の情報を受け取って、動画台本を作成してください。

```text
動画タイプ：ゆっくり解説 / ずんだもん解説 / 自動選択
テーマ：{{THEME}}
想定尺：{{DURATION}}
想定視聴者：{{AUDIENCE}}
動画の目的：{{GOAL}}
トーン：{{TONE}}
必ず入れたい内容：{{MUST_INCLUDE}}
避けたい内容：{{AVOID}}
参考資料：{{REFERENCES}}
```

未入力の項目がある場合は、最も自然な前提で補完してください。
ただし、事実確認が必要な内容は推測で断定しないでください。

## キャラクター構成

### ゆっくり解説の場合

- 霊夢：視聴者代表。疑問、驚き、ツッコミ、感情的リアクションを担当。
- 魔理沙：解説役。知識を整理して説明する担当。

基本比率：

- 魔理沙：60〜70%
- 霊夢：30〜40%

ただし、霊夢が質問し、魔理沙が説明するだけの単調な流れにしないでください。
霊夢にも仮説、誤解、反論、共感、ツッコミを入れて、会話として自然にしてください。

### ずんだもん解説の場合

- ずんだもん：視聴者代表。素直な疑問、ボケ、驚き、感情的リアクションを担当。
- 四国めたん：解説役。整理、補足、冷静なツッコミを担当。

基本比率：

- 四国めたん：55〜65%
- ずんだもん：35〜45%

ずんだもんは語尾に「なのだ」を使ってもよいですが、使いすぎて不自然にならないようにしてください。

## 台本の基本構成

以下の構成で作成してください。

```text
1. 冒頭フック
2. 挨拶
3. テーマ提示
4. 視聴者が気になる疑問の提示
5. お品書き・章立て
6. 第1章：前提・基礎知識
7. 第2章：核心・仕組み・理由
8. 第3章：具体例・比較・失敗例・意外な話
9. 第4章：結論・実用的な学び
10. まとめ
11. コメント誘導・次回予告
12. 締め
```

想定尺が短い場合は、章数を3章程度に圧縮してください。
想定尺が長い場合は、4〜6章まで増やしても構いません。

## 参照動画型プロファイル

ユーザーが「参照動画みたいに」「未解明の謎」「事件」「疑惑」「長尺ゆっくり解説」のような依頼をした場合は、`reference_style_profile.md` を併用してください。

`reference_style` は次の4つから1つ選びます。

- `listicle_mystery`: 未解明の謎○選型
- `scandal_case`: 疑惑・炎上・噂紹介型
- `crime_timeline`: 事件経緯・人物像深掘り型
- `deep_mystery`: 人類・歴史・科学ミステリー深掘り型

この型を使う場合、各シーンには次の参照再現メタを必ず入れてください。

```yaml
reference_style: listicle_mystery
reference_beat: numbered_case
hook_type: unresolved_question
curiosity_gap: なぜ今も正体が分からないのか
evidence_role: background
next_reason: 次の事例では別の未解明点を見る
```

中盤40〜60%地点には `reference_beat: midpoint_rehook` を入れ、終盤には `reference_beat: summary` を入れて冒頭の疑問へ戻ってください。
事件・疑惑・実在人物を扱う場合は、序盤に `reference_beat: disclaimer` を置き、断定できない情報を「噂」「指摘」「可能性」「断定はできない」と明示してください。

## 冒頭フックのルール

冒頭5〜10秒で、視聴者が続きを見たくなる一言を入れてください。

良いフックの例：

```text
霊夢：え、これって普通に使ってる人多いけど、実はかなり危ないってこと？
魔理沙：そうだぜ。知らないままだと、損するどころか普通に詰む可能性もあるんだ。
```

```text
ずんだもん：なんとなく便利そうって思ってたけど、そんな落とし穴があるのだ？
めたん：あるわ。しかも、多くの人が一番大事な部分を見落としているの。
```

冒頭で結論をすべて言い切らず、続きを見たくなる疑問を残してください。

## 面白い台本にするルール

単なる説明ではなく、以下を必ず入れてください。

- 視聴者が誤解しやすいポイント
- 「実は〇〇」という意外性
- 具体例
- 失敗例
- たとえ話
- キャラ同士の軽いツッコミ
- 最後に持ち帰れる結論

説明が長くなる場合は、霊夢・ずんだもん側に疑問やツッコミを挟ませてテンポを作ってください。

### 視聴維持の最低条件

- 冒頭5秒以内に、損失、意外性、誤解訂正、または強い疑問を入れてください。
- 1シーン1役割にし、同じ結論文や同じ注意喚起を繰り返さないでください。
- 解説役だけが3セリフ以上続く独演を避けてください。
- 3〜5シーンごとに、失敗例、比較、手順、反論、まとめなど視点を切り替えてください。
- 最後は抽象的な感想ではなく、視聴者が今日やる具体行動で終えてください。

## 台詞の書き方

### 基本ルール

- 1セリフは短めにしてください。
- 1セリフで複数の論点を詰め込まないでください。
- 難しい用語は、初出時に会話内でかみ砕いて説明してください。
- 書き言葉ではなく、聞いて自然な話し言葉にしてください。
- 説明役だけが長く話し続けないようにしてください。

### 悪い例

```text
魔理沙：AI動画生成とは、生成AI技術を活用して画像、音声、字幕、構成要素を統合的に処理し、従来の動画編集工程を効率化する手法であり、近年ではSNSマーケティングや教育コンテンツ制作にも活用されています。
```

### 良い例

```text
魔理沙：AI動画生成っていうのは、ざっくり言うと、台本や画像や音声をAIに手伝わせて動画を作る方法だな。
霊夢：つまり、人間が全部手作業で編集しなくてもいいってこと？
魔理沙：そうだぜ。ただし、完全放置で神動画ができるわけじゃない。
```

## 素材候補メモの扱い

台本生成時点では、編集演出ではなく「どんな素材が必要そうか」だけをメモしてください。

例：

```text
素材候補：AIツールの画面イメージ、PC作業中のイラスト、比較表、注意マーク
```

以下のような編集指定はしないでください。

```text
NG：ここで画像をズームイン、SEドン、字幕を黄色にする
```

素材候補は、後続の `02_演出編集プロンプト.md` で具体的な演出に変換します。

動画化用の素材メモでは、各シーンに次も入れてください。

### 1動画1テンプレート固定ルール

- `Scene01`〜`Scene21` のテンプレートは、動画全体の見た目スキンです。
- `scenes[]` は時間ブロックです。シーンごとにテンプレートを切り替えないでください。
- 動画化用 `script.yaml` では `meta.layout_template` に1つだけ指定します。
- `scenes[].scene_template` は書かないでください。
- 各 scene は `main` / `sub` / `title_text` / `dialogue` の中身だけを変えてください。

```yaml
scene_goal: このシーンで理解させること
viewer_question: 視聴者役が抱く疑問
visual_role: main枠とsub枠が何を担当するか
reference_style: listicle_mystery / scandal_case / crime_timeline / deep_mystery
reference_beat: op_hook / disclaimer / numbered_case / timeline / evidence / counterpoint / unresolved_point / midpoint_rehook / summary / cta
hook_type: unresolved_question / strange_fact / rumor_or_suspicion / timeline_gap / counterintuitive / loss_or_warning / none
curiosity_gap: 視聴者が次に知りたくなる未回収の疑問
evidence_role: claim / background / evidence / counterpoint / limit / interpretation / summary / none
next_reason: 次のシーンを見る理由
visual_asset_plan:
  - slot: main / sub / chapter / comparison / summary
    purpose: この素材で理解させること
    insert_timing: どのセリフの前後に入れるか
    asset_path: script/{episode_id}/assets/s01_main.png
    imagegen_prompt: シーン目的、動画トーン、テンプレート枠、構図、色、余白、禁止要素、文字有無を含む具体プロンプト
    audit_points:
      - 1枚1メッセージ
      - テンプレート枠に収まる
      - 細かい文字やロゴがない
asset_requirements:
  description: 画像に入れる内容
  imagegen_prompt: テンプレート枠に入る図解としての具体プロンプト
typography:
  subtitle_family: gothic / mincho
  content_family: gothic / mincho
  title_family: gothic / mincho
```

画像生成プロンプトは、1枚1メッセージ、中央主題、余白あり、細かい文字なしを基本にしてください。
`Scene02` / `Scene03` / `Scene10` / `Scene13` / `Scene14` のようにサブ枠がある場合、mainは大きな主題、subは3項目以内の補足に分けてください。
本編シーンは原則として最低1枚の main 画像を要求してください。長尺動画では45〜60秒ごとに最低1つ、新しい視覚素材を置いてください。
`imagegen_prompt` は「フラットな図解」「わかりやすいカード」だけで終わらせず、動画の雰囲気、テンプレート枠、構図、色、余白、禁止要素、文字を入れるか入れないかまで書いてください。
フォント分類は `gothic` / `mincho` のみを使ってください。ゴシックを明示する場合は、レンダー側で `public/fonts/keifont.ttf` のけいふぉんとを使う前提です。

## 品質チェック

出力後、以下を自己チェックしてください。

- 霊夢と魔理沙、またはずんだもんとめたんの役割が分かれているか
- 解説役だけが長く話しすぎていないか
- 冒頭に続きを見たくなるフックがあるか
- テーマを渡すだけでも成立する構成になっているか
- 章立てが自然か
- 難しい用語をかみ砕いているか
- 素材候補メモはあるが、編集演出まで踏み込んでいないか
- 最後に視聴者が得られる結論があるか
- 同じセリフや同じ箇条書きが複数シーンで使い回されていないか
- 各シーンに `scene_goal`、`viewer_question`、`visual_role` があるか
- 画像素材に具体的な `asset_requirements.imagegen_prompt` があるか
- 台本内容に応じて `typography` が `gothic` / `mincho` で指定されているか
- 各シーンに `visual_asset_plan` があるか
- 本編シーンに十分な main 画像があり、45〜60秒以上新しい視覚素材が空いていないか
- `imagegen_prompt` が動画トーン、テンプレート枠、構図、余白、禁止要素まで具体化されているか
