# 参照動画型 台本プロファイル

## 使うタイミング

ユーザーが「参照動画みたいに」「ミステリー系」「事件系」「疑惑系」「長尺ゆっくり解説」と依頼した場合、このプロファイルを必ず使う。

テンプレート選択とは別に、台本の型として `reference_style` を1つ選ぶ。

- `listicle_mystery`: 未解明の謎○選型
- `scandal_case`: 疑惑・炎上・噂紹介型
- `crime_timeline`: 事件経緯・人物像深掘り型
- `deep_mystery`: 人類・歴史・科学ミステリー深掘り型

## 共通ルール

- 冒頭30秒以内に、謎、疑惑、違和感、意外な事実を置く。
- 本編は「番号付きリスト」または「時系列深掘り」に寄せる。
- 各章は `謎提示 -> 背景 -> 証拠/根拠 -> 反論/限界 -> 未解明点/次の疑問` で進める。
- 中盤40〜60%地点に `midpoint_rehook` を入れる。
- 終盤は最初の謎に戻って回収する。
- 事件・疑惑・実在人物を扱う場合、断定回避と注意書きを必ず入れる。

## script.yaml メタ出力

各シーンに次を必ず入れる。

```yaml
reference_style: listicle_mystery
reference_beat: numbered_case
hook_type: unresolved_question
curiosity_gap: なぜ今も正体が分からないのか
evidence_role: background
next_reason: 次の事例では伝承と現実の境目を見る
```

## style別の構成

### listicle_mystery

```text
OP -> 挨拶 -> テーマ提示 -> 1つ目 -> 背景 -> 仮説 -> 未解明点 -> 次項目 -> まとめ -> ED
```

必須:

- `numbered_case` を最低3回。
- 各項目に `curiosity_gap`。
- 最後に「共通して残る謎」をまとめる。

### scandal_case

```text
OP小芝居 -> テーマ提示 -> 注意書き -> ケース紹介 -> 疑惑点 -> 反論/限界 -> 現在の扱い -> 次ケース -> まとめ
```

必須:

- `disclaimer` を序盤に入れる。
- `evidence_role` に `counterpoint` と `limit` を含める。
- 「噂」「指摘」「可能性」「断定はできない」を必要箇所に入れる。

### crime_timeline

```text
OP -> 事件概要 -> 舞台/人物 -> 当日の流れ -> 分岐点 -> その後 -> 残る疑問 -> 注意喚起/風化防止
```

必須:

- `timeline` を複数入れる。
- 目的は注意喚起、再発防止、風化防止。
- 被害者や関係者を茶化さない。

### deep_mystery

```text
OP -> 大きな疑問 -> 前提整理 -> 昔はどうだったか -> なぜ変わったか -> 他説/反論 -> 今に残る影響 -> 結末の問い
```

必須:

- 大きな問いは1つに絞る。
- 中盤で見方を反転させる。
- 終盤で最初の問いに戻る。

## 自己監査

生成後、次を確認する。

- 冒頭2シーン以内に強い `hook_type` がある。
- 80%以上のシーンに参照メタが入っている。
- `midpoint_rehook` と `summary` がある。
- 視聴者役の疑問、驚き、言い換えが十分にある。
- 画像計画が1画面1情報になっている。
- 断定回避が必要なテーマで、注意書きと限界説明がある。
