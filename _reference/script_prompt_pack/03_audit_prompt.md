# 03_audit_prompt

## 目的

生成済み台本を監査し、動画化してよいか判定する。
必ず `00_MASTER_SCRIPT_RULES.md` と対象台本を読む。

## 入力

```text
対象台本：
使用テンプレート：
キャラペア：
想定尺：
```

## 監査観点

- 冒頭5秒が弱くないか。
- 各シーン冒頭1から2セリフに小さなフックがあるか。
- キャラの役割が崩れていないか。
- 解説役が独演していないか。
- リアクション役が相槌だけになっていないか。
- 各シーンにボケ→ツッコミ、または誤解→訂正があるか。
- 会話がQ&Aだけで進んでいないか。
- L3以上のリアクションが動画全体で2回以上あるか。
- 「なのだ」「のだ」などの口癖が過剰、または少なすぎないか。
- 想定尺に対して本編シーン数、本編セリフ数、1シーン平均が足りているか。
- 5分前後で10シーン以上、90セリフ以上、平均8セリフ以上を満たすか。
- 5分前後で8シーン以下、70セリフ未満になっていないか。
- 各シーンが6セリフ前後で固定され、単調になっていないか。
- メタ情報で水増しされ、本編会話が薄くないか。
- 1セリフが25文字以内か。
- 各本編シーンに数字、具体例、あるあるのいずれかがあるか。
- 章タイトルがフック型で、説明目次になっていないか。
- `scene_format` が3種類以上あるか。
- 例え話または小ネタが動画全体で2回以上あるか。
- 同じセリフ、同じ結論、同じ箇条書きが反復していないか。
- 中盤40から60パーセント地点に再フックがあるか。
- 最後が具体行動で終わっているか。
- 最終シーンが「見るだけ」で終わっていないか。
- 最終行動に確認、選択、保存/逃がす/テンプレ化、予定化など2アクション以上があるか。
- 章タイトルにテーマ外の単語が混ざっていないか。
- 章タイトルが本文内容とズレていないか。
- 誤字っぽい語尾、助詞抜け、不自然な口語が残っていないか。
- RMでは魔理沙の「だぜ」比率が過剰または不足していないか。
- 監査が甘く、改善余地ゼロ扱いになっていないか。
- テンプレートの main / sub / subtitle / title / avoid_area に合っているか。
- 全本編シーンに `visual_asset_plan` があるか。
- 全画像に `image_direction`、`supports_dialogue`、`supports_moment`、`visual_type`、`composition_type` があるか。
- `imagegen_prompt` が具体的か。
- `imagegen_prompt` がGPT-Image-2向け完成形式を満たしているか。
- `imagegen_prompt` に scene_id、slot、main/sub の役割、字幕帯とキャラ位置を避ける余白があるか。
- `imagegen_prompt` に、台本内のどの掛け合いを補強するか、前景/中景/背景、色、光、Remotion重ね文字スペース、禁止要素があるか。
- main枠とsub枠の画像役割が分かれているか。
- 画像内文字が0文字または日本語3〜6文字以内に抑えられているか。
- episode共通の `meta.image_style` と各シーンの `imagegen_prompt` の絵柄が揃っているか。
- 「フラットな図解」「わかりやすいカード」「中央に主題、余白多め」だけの抽象プロンプトになっていないか。
- `meta.layout_template` が `Scene01` から `Scene21` 形式か。
- `scenes[].scene_template` を使っていないか。
- 参照動画型では `reference_style`、`reference_beat`、`hook_type`、`curiosity_gap`、`evidence_role`、`next_reason` があるか。
- 事件、疑惑、実在人物では断定回避、注意書き、限界説明があるか。

次の表で100点満点を付ける。

|---|---:|---|---|---|
| 冒頭フック | 15 |  |  |  |
| ボケ→ツッコミのリズム | 20 |  |  |  |
| リアクション強度 | 15 |  |  |  |
| 数字・具体性 | 10 |  |  |  |
| 章タイトルのフック力 | 10 |  |  |  |
| 例え話・小ネタ | 10 |  |  |  |
| キャラ立ち | 10 |  |  |  |
| シーン形式のバリエーション | 10 |  |  |  |

判定基準:

- 80点以上: PASS。ただし尺・密度チェックでFAILがある場合はPASS禁止。
- 65から79点: 仮PASS、改善推奨
- 64点以下: FAIL

## 尺・密度チェック

想定尺と本編会話量を確認する。メタ情報は尺として数えない。

| 項目 | 基準 | 結果 | 判定 |
|---|---|---|---|
| 想定尺 | 入力と一致 |  |  |
| 本編シーン数 | 5分前後なら10以上 |  |  |
| 本編セリフ数 | 5分前後なら90以上 |  |  |
| 1シーン平均 | 5分前後なら8以上 |  |  |
| 6セリフ固定化 | 単調ならNG |  |  |
| 中盤再フック | 40〜60%地点にある |  |  |
| メタ水増し | 本編会話が薄くない |  |  |

5分指定で本編セリフ数が70未満の場合、原則FAIL。
5分指定で本編シーン数が8以下の場合、原則FAIL。
尺・密度チェックがFAILなら、ジャンル適合スコアが高くても最終PASSは禁止。

## 最終行動チェック

| 項目 | 基準 | 結果 | 判定 |
|---|---|---|---|
| 行動の具体性 | 視聴後すぐ実行できる |  |  |
| 2アクション以上 | 確認＋選択＋保存/整理/予定化のうち2つ以上 |  |  |
| 見るだけで終わらない | 確認後の次手がある |  |  |

## Blocking Issues の基準

次が1つでもある場合は Blocking Issues として扱う。

- 視聴者代表キャラが聞き役だけになっている。
- 章タイトルが説明目次になっている。
- 章タイトルにテーマ外の単語が混ざっている。
- 数字・具体例が全体的に不足している。
- 会話がQ&Aだけで進んでいる。
- 「なのだ」「のだ」などの口癖が過剰、またはZMで10%未満。
- 解説役が独演している。
- 各シーンにボケ→ツッコミ、または誤解→訂正がない。
- L3以上のリアクションが動画全体で2回未満。
- シーン形式が3種類未満。
- `visual_asset_plan` または `imagegen_prompt` が欠けている。
- `image_direction`、`supports_dialogue`、`supports_moment`、`visual_type`、`composition_type` が欠けている。
- `imagegen_prompt` のGPT-Image-2向け完成形式が欠けている。
- `imagegen_prompt` にテンプレート枠、色、情報量、禁止要素のいずれかがない。
- `imagegen_prompt` が白背景中央アイコン、汎用素材、会話と無関係な図解になりそう。
- 画像内文字が長文、英語UI、細かい表、ブランドロゴ、実在UI模写を要求している。
- sub枠に複雑な図や長文を要求している。
- YAML化前提の構造ルールを壊している。
- 想定尺に対して本編セリフ数が不足している。
- 5分指定なのに本編70セリフ未満。
- 5分指定なのに8シーン以下。
- メタ情報で水増しされ、本編会話が薄い。
