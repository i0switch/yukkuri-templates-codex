# 00_MASTER_SCRIPT_RULES

## 最重要ソース

台本品質は次のローカル正本を最上位にする。

- `_reference/script_prompt_pack/local_canonical/yukkuri_master.md`
- `_reference/script_prompt_pack/local_canonical/zundamon_master.md`

どちらかが読めない場合は推測で進めず、読めなかったファイル名と必要な対応を報告して停止する。

## 基本方針

- 出力は日本語。
- 「今回は〇〇について解説します」から始めない。
- 冒頭5秒で損失、意外性、誤解訂正、強い疑問、数字のいずれかを置く。
- 1本の動画で使うテンプレートは `Scene01`〜`Scene21` の1つだけ。
- テンプレートは `meta.layout_template` にのみ書く。
- `scenes[].scene_template` と新規 `meta.scene_template` は使わない。
- 台本は説明文ではなく、ボケ、勘違い、雑な解決策、ツッコミ、具体例、結論で進む会話にする。
- 不安を煽るだけで終わらせず、必ず対処法と次の行動へ落とす。
- BGM、SE、細かい編集演出はRemotion側のテンプレート (`Scene01`〜`Scene21`) で吸収する。本パックでは扱わない。

## 必須入力

- テーマ
- ざっくり入れたい内容
- 想定尺
- 使用テンプレート
- キャラペア: `RM` または `ZM`

不足がある場合は `01_input_normalize_prompt.md` で保守的に補う。ただしテンプレート未指定の場合は、テンプレート選択を先に確定する。

## 尺と密度

| 想定尺 | シーン数 | セリフ数 | 画像目安 |
|---|---:|---:|---:|
| 3分 | 6〜8 | 60〜80 | 6〜8 |
| 5分 | 10〜12 | 90〜130 | 10〜12 |
| 8分 | 14〜18 | 150〜220 | 14〜18 |

FAIL条件:

- 5分で90セリフ未満。
- 5分で10シーン未満。
- 3分で60セリフ未満。
- 3分で6シーン未満。
- 解説役が3セリフ以上連続する。
- 視聴者代表キャラが質問か相槌だけ。
- 章タイトルが「〇〇とは」「〇〇について」ばかり。
- 中盤40〜60%地点に再フックがない。
- 最後が具体行動で終わらない。

## 会話品質

各本編シーンに最低1回、次の流れを入れる。

1. 視聴者代表キャラが浅い理解、勘違い、雑な解決策、極端な発想、怖がり、疑いのどれかを出す。
2. 解説役が短くツッコむ、または訂正する。
3. 数字、具体例、失敗例、あるある、比較のどれかで補強する。
4. 視聴者代表キャラが驚く、疑う、言い換える、ボケる。
5. 解説役が短く結論を置く。

禁止:

- Q&Aだけで進む会話。
- 解説役の独演。
- 「なるほど」「そうなのだ」「え？」だけの反応。
- 箇条書き説明の読み上げ。

## RM: 霊夢・魔理沙

- 霊夢は視聴者代表。詳しすぎない。
- 霊夢は質問だけでなく、勘違い、ボケ、雑な理解、怖がり、ツッコミを担当する。
- 魔理沙は解説役。短いツッコミ、具体例、数字、結論を出す。
- 魔理沙の「だぜ」は自然に使い、連発しない。
- 比率は霊夢40、魔理沙60を目安にする。
- 魔理沙だけが3セリフ以上連続しない。
- 霊夢のL3以上リアクションを複数回入れる。

## ZM: ずんだもん・四国めたん

- ずんだもんは視聴者代表。賢くしすぎない。
- ずんだもんは質問だけでなく、勘違い、ボケ、雑な理解、焦り、ツッコミを担当する。
- 「なのだ」「のだ」は自然に混ぜる。全セリフに付けない。
- 語尾は「なのだ」「のだ」「だね」「かな」「じゃん」「マジで？」などを混ぜる。
- めたんは冷静に刺す。短く訂正して具体例を出す。
- 比率はずんだもん40、めたん60を目安にする。
- めたんだけが3セリフ以上連続しない。
- ずんだもんの反応を「そうなのだ」だけにしない。

## リアクション強度

| 強度 | 役割 | 例 |
|---|---|---|
| L1 | 軽い相槌 | なるほど / そうなんだ |
| L2 | 素朴な疑問 | え、なんで？ / どういうこと？ |
| L3 | 強い驚き | マジで！？ / それ普通にヤバい |
| L4 | 焦り・呆れ | それは詰む / 完全に油断してた |

1本の台本にL3以上を最低2回入れる。

## 章タイトル

説明目次ではなくフック型にする。

推奨:

- 写真を消す前に見る場所
- バックアップ済みの罠
- 容量不足で詰む瞬間
- 安全に逃がす3ステップ
- ふっくらを壊す冷凍の罠

禁止:

- 〇〇とは
- 〇〇について
- 理由を解説
- 方法を紹介

## テンプレート対応

必ず選択テンプレート (`Scene01`〜`Scene21`) のレイアウト仕様を読み、次を確定する。

- `main_content`
- `sub_content` または `sub_contentなし`
- `subtitle_area`
- `title_area` または `title_areaなし`
- `character_layout`
- `avoid_area`
- `subtitle_text_required`

サブ枠ありの `Scene02` / `Scene03` / `Scene10` / `Scene13` / `Scene14` では、subをmainと別役割で使う。

subの役割:

- 3項目チェック
- NG / OK 比較
- 注意点
- 数字の補足
- 手順の現在地
- 次にやること
- ボケ・ツッコミ補助

sub枠なしテンプレートでは `sub: null` にし、無理にsub素材を作らない。

字幕が狭いテンプレートでも `dialogue[].text` は自然な発話単位を維持する。
字幕の折り返し、縮小、収まりはRemotion側の描画で調整する。

## 画像プロンプト

ChatGPT Images 2.0を前提にする。

各シーンに `visual_asset_plan` を作り、`imagegen_prompt` には `script_final.md` の対象シーン全文をそのまま入れる。

画像は会話字幕とは別に出すコンテンツ挿入画像であり、アスペクト比は16:9にする。

作風はLLMが `script_final.md` 全体を読んで判断する。

`image_direction`、`visual_type`、`supports_dialogue`、`supports_moment`、`composition_type`、`hook_type`、`myth_vs_fact`、`boke_or_reaction` などの抽象タグは v2 では使用しない。`imagegen_prompt` 本文にも混ぜない。

禁止:

- 白背景中央アイコン。
- 汎用フラット図解。
- 既存キャラ生成。
- 実在人物。
- ブランドロゴ。
- 実在UIの模写。
- 8枚グリッド、一括生成、sprite sheet、asset sheet、crop、切り出し。
- 1つの codex exec プロセスに複数画像のプロンプトをまとめて渡す（1プロセス=1画像厳守）。

## YAML

- `meta.layout_template`: `Scene01`〜`Scene21`
- `meta.pair`: `RM` / `ZM`
- `scenes[].scene_template`: 使用禁止
- 画像: `main.asset` / `sub.asset`
- asset: `assets/s01_main.png`
- subなし: `sub: null`
- 字幕: `dialogue[].text`
- `script_final.md` の自然な発話単位を維持
- 表示都合の機械分割は禁止
- `main.kind` は `image` のみ。`sub.kind` は `image` / `text` / `bullets` / `null` のいずれか
- `main.caption` / `main.text` / `main.items` は使わない。`sub` はテキスト・箇条書きを使ってよい
- 字幕の折り返し・収まりはRemotion側で吸収する
- `bgm:` ブロックは必須
- `meta.width × meta.height` のデフォルトは FullHD（1920×1080）。レイアウトは 1920×1080 base から自動スケール

## 監査

PASSでも軽微な改善余地を1つ以上書く。

必ずFAIL:

- 5分で90セリフ未満。
- 5分で10シーン未満。
- 3分で60セリフ未満。
- 霊夢が質問だけ。
- ずんだもんが「そうなのだ」系ばかり。
- 魔理沙またはめたんの独演が多い。
- 章タイトルが説明目次。
- 画像プロンプトが抽象的。
- テンプレートのsub / subtitle / title / avoid_areaを考慮していない。
- YAML化で台本の面白さが消えている。
