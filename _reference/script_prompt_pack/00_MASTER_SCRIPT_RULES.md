# 00_MASTER_SCRIPT_RULES

## 最重要ソース

台本品質は次のローカル正本を最上位にする。

- `_reference/script_prompt_pack/local_canonical/yukkuri_master.md`
- `_reference/script_prompt_pack/local_canonical/zundamon_master.md`

どちらかが読めない場合は推測で進めず、読めなかったファイル名と必要な対応を報告して停止する。

## 基本方針

- 出力は日本語。
- 「今回は〇〇について解説します」から始めない。
- 冒頭1〜3発話では、いきなり結論・危険ワードだけに入らない。
- 冒頭1〜3発話で「日常の状況」「視聴者がやりがちな行動」「なぜ今この話をするか」のどれかを置き、視聴者の現在地を作る。
- 冒頭5秒で損失、意外性、誤解訂正、強い疑問、数字のいずれかを置く。
- 冒頭1番目のセリフは、損失提示、衝撃数字、誤解訂正、強い疑問、あるあるからの意外性のどれかにする。「あるある」だけ、「今回は〜について」、「こんにちは」、「〇〇って知ってる？」だけの冒頭は禁止する。
- 日常の前提を置いた後に、損失、危険性、誤解訂正へ入る。
- 1本の動画で使うテンプレートは `Scene01`〜`Scene21` の1つだけ。
- テンプレートは `meta.layout_template` にのみ書く。
- `scenes[].scene_template` と新規 `meta.scene_template` は使わない。
- 台本は説明文ではなく、ボケ、勘違い、雑な解決策、ツッコミ、具体例、結論で進む会話にする。
- 不安を煽るだけで終わらせず、必ず対処法と次の行動へ落とす。
- BGM、SE、字幕強調、細かい編集演出はRemotion側で描画・再生する。ただし本パックでは、`motion_mode` と `dialogue[].emphasis` として「どこを刺すか」の最低限の演出メタを渡す。

## 冒頭15秒の維持率フック

s01 は必ず次の順番で始める。

1. 0〜5秒: 損失、数字、意外性、誤解訂正、強い疑問のどれかを置く。
2. 5〜10秒: 視聴者が「自分のことかも」と思うあるあるを置く。
3. 10〜15秒: この動画を見る理由を1文で置く。

禁止:

- 「今回は〇〇について解説します」。
- テーマ説明だけで始める。
- 「大事です」「危険です」だけで煽る。
- 数字を出しても生活被害に接続しない。

## 必須入力

- テーマ
- ざっくり入れたい内容
- 想定尺
- 使用テンプレート
- キャラペア: `RM` または `ZM`

不足がある場合は `01_input_normalize_prompt.md` で保守的に補う。ただしテンプレート未指定の場合は、テンプレート選択を先に確定する。

## 尺と密度

| 想定尺 | シーン数 | セリフ数 | 画像目安 |
|---|---:|---:|---:|
| 3分 | 6〜8 | 60〜80 | 6〜8 |
| 5分 | 10〜12 | 90〜130 | 10〜12 |
| 8分 | 14〜18 | 150〜220 | 14〜18 |

5分動画の最終判断は、発話数だけではなくTTSエンジン別の推定自然音声尺で行う。

- RM / AquesTalk: 初期係数 `5.2秒/発話`
- ZM / VOICEVOX: 初期係数 `3.8秒/発話`
- `target_duration_sec: 300` の許容範囲は `270〜330秒`
- 音声速度変更は禁止。短すぎる場合は台本量を増やし、長すぎる場合は自然尺を優先してそのまま使う
- 尺不足時は `07_rewrite_prompt.md` で不足分だけ `script_final.md` を台本補完し、再レビューしてから進む

FAIL条件:

- 5分で推定自然音声尺が下限未満。
- 5分で10シーン未満。
- 3分で60セリフ未満。
- 3分で6シーン未満。
- 解説役が3セリフ以上連続する。
- 視聴者代表キャラが質問か相槌だけ。
- RMでは、1シーン内の霊夢の疑問形は最大1個。質問が続く場合は、ボケ、誤解、強い反応、ツッコミ、行動宣言、実体験に置き換える。
- RMでは、魔理沙の「だぜ」末尾は全rightセリフの30〜60%に収め、2セリフ連続で「だぜ」末尾にしない。
- 章タイトルが「〇〇とは」「〇〇について」ばかり。
- 中盤40〜60%地点に再フックがない。
- 最後が具体行動で終わらない。

## 会話品質

各本編シーンに最低1回、次の流れを入れる。

1. 視聴者代表キャラが浅い理解、勘違い、雑な解決策、極端な発想、怖がり、疑いのどれかを出す。
2. 解説役が短くツッコむ、または訂正する。
3. 数字、具体例、失敗例、あるある、比較のどれかで補強する。
4. 視聴者代表キャラが驚く、疑う、言い換える、ボケる。
5. 解説役が短く結論を置く。

各本編シーンの視聴者代表キャラの1発目は、単なる質問ではなく、勘違い、言い訳、視聴者の本音、雑な解決策、怖がり、疑い、的外れなたとえ、自分に都合のいい解釈のどれかにする。

解説役は説明に入る前に、必ず短いツッコミまたは訂正を入れる。

各シーンに最低1つ、視聴者が経験していそうな「あるある」を入れる。

各シーンに最低1つ、金額、時間、回数、速度、年数、手順数、比較、実例、失敗例のいずれかを入れる。数字が出せない場合は、具体的な生活場面を入れる。

禁止:

- Q&Aだけで進む会話。
- 解説役の独演。
- 「なるほど」「そうなのだ」「え？」だけの反応。
- 箇条書き説明の読み上げ。

## 中盤のダレ防止

動画全体の40〜60%地点に再フックを入れる。

再フックでは最低2つ入れる。

- 視聴者の誤解をひっくり返す。
- 「実は犯人は〇〇」型の言い切り。
- 強い数字。
- 失敗例。
- 視聴者代表キャラのL3以上リアクション。
- 解説役の短い断言。

中盤シーンでは、衝撃数字の再提示、失敗例の生々しい描写、視聴者煽り、比較のいずれかを最低1つ入れる。

## オチとコメント誘導

ラストは感想で終わらせない。必ず「今日やる行動」を1つに絞って終える。

ラスト30秒以内に、視聴者が答えやすいコメント誘導を1つ入れる。コメント誘導は説教ではなく、あるある共有にする。
最後から2番目のシーンには、投票形式の問いかけ、共感誘発のあるある、強めの言い切りのいずれかを入れ、最終行動への温度を上げる。

## RM: 霊夢・魔理沙

- 霊夢は視聴者代表。詳しすぎない。
- 霊夢は質問だけでなく、勘違い、ボケ、雑な理解、怖がり、ツッコミを担当する。
- 魔理沙は解説役。短いツッコミ、具体例、数字、結論を出す。
- 魔理沙の「だぜ」は自然に使い、連発しない。
- 「するだぜ」「使うだぜ」「行くだぜ」「あるあるだぜ」「危険だぜ」「古さも見るだぜ」「作ろうだぜ」のような不自然な連結は禁止。
- 比率は霊夢40、魔理沙60を目安にする。
- 魔理沙だけが3セリフ以上連続しない。
- 霊夢のL3以上リアクションを複数回入れる。

## ZM: ずんだもん・四国めたん

- ずんだもんは視聴者代表。賢くしすぎない。
- ずんだもんは質問だけでなく、勘違い、ボケ、雑な理解、焦り、ツッコミを担当する。
- 「なのだ」「のだ」は自然に混ぜる。全セリフに付けない。
- 語尾は「なのだ」「のだ」「だね」「かな」「じゃん」「マジで？」などを混ぜる。
- めたんは冷静に刺す。短く訂正して具体例を出す。
- 比率はずんだもん40、めたん60を目安にする。
- めたんだけが3セリフ以上連続しない。
- ずんだもんの反応を「そうなのだ」だけにしない。

## リアクション強度

| 強度 | 役割 | 例 |
|---|---|---|
| L1 | 軽い相槌 | なるほど / そうなんだ |
| L2 | 素朴な疑問 | え、なんで？ / どういうこと？ |
| L3 | 強い驚き | マジで！？ / それ普通にヤバい |
| L4 | 焦り・呆れ | それは詰む / 完全に油断してた |

1本の台本にL3以上を最低2回入れる。

## 章タイトル

説明目次ではなくフック型にする。

推奨:

- 写真を消す前に見る場所
- バックアップ済みの罠
- 容量不足で詰む瞬間
- 安全に逃がす3ステップ
- ふっくらを壊す冷凍の罠

禁止:

- 〇〇とは
- 〇〇について
- 理由を解説
- 方法を紹介

## テンプレート対応

必ず選択テンプレート (`Scene01`〜`Scene21`) のレイアウト仕様を読み、次を確定する。

- `main_content`
- `sub_content` または `sub_contentなし`
- `subtitle_area`
- `title_area` または `title_areaなし`
- `character_layout`
- `avoid_area`
- `subtitle_text_required`

サブ枠ありの `Scene02` / `Scene03` / `Scene10` / `Scene13` / `Scene14` では、subをmainと別役割で使う。
全シーンで `sub.kind: text` または `sub.kind: bullets` を必須にし、`sub: null` や `sub.kind: image` にはしない。

subの役割:

- 4〜6項目チェック（推奨6項目）
- NG / OK 比較
- 注意点
- 数字の補足
- 手順の現在地
- 次にやること
- ボケ・ツッコミ補助

subの行数:

- `sub.kind: bullets` は原則4〜6項目にする。情報量があるシーンでは6項目を推奨する。
- `sub.kind: text` は4〜6行の短い補足文にし、改行区切りで読ませる。
- 3項目だけのsubは、締め・要約・最終行動など情報を絞る意図が明確な場合だけ使う。
- 各項目は短くし、会話字幕の本文をそのまま複製しない。

sub枠なしテンプレートでは `sub: null` にし、無理にsub素材を作らない。

字幕が狭いテンプレートでも `dialogue[].text` は自然な発話単位を維持する。
字幕の折り返し、縮小、収まりはRemotion側の描画で調整する。

## 画像プロンプト

ChatGPT Images 2.0を前提にする。

各シーンに `visual_asset_plan` を作り、`imagegen_prompt` には `script_final.md` の対象シーン全文をそのまま入れる。

画像は会話字幕とは別に出すコンテンツ挿入画像であり、アスペクト比は16:9にする。

作風はLLMが `script_final.md` 全体を読んで判断する。

`visual_type`、`supports_dialogue`、`supports_moment`、`hook_type`、`myth_vs_fact`、`boke_or_reaction` などの旧抽象タグは v2 では使用しない。`imagegen_prompt` 本文にも混ぜない。

各シーンの画像には、`image_role` と `composition_type` を必ず指定する。

- `image_role`: 理解補助 / 不安喚起 / 笑い / 比較 / 手順整理 / 証拠提示 / オチ補助
- `composition_type`: NG / OK 比較 / 失敗例シミュレーション / 誇張図解 / 証拠写真風 / チェックリスト / 手順図 / 原因マップ / ビフォーアフター / ツッコミ待ち構図 / 事故寸前構図

3シーン連続で同じ `composition_type` を使ってはいけない。

画像プロンプトには、画面で一番大きく見せる対象、その理由、小さくする/入れない対象、画像化する一言、画像で再現する状況、画像で再現しない要素を入れる。

対象シーンタイトルは、画像内の大きく目立つ見出しとして必ず入れる。
例: `s01: 月580円が年6,960円に化ける` の場合、画像内に入れる文字は `月580円が年6,960円に化ける` だけにし、`s01` は入れない。
会話全文は引き続き画像内に並べない。

字幕スペースは別途用意されているため、画像では絶対に用意しない。

禁止:

- 白背景中央アイコン。
- 汎用フラット図解。
- 既存キャラ生成。
- 実在人物。
- ブランドロゴ。
- 実在UIの模写。
- 8枚グリッド、一括生成、sprite sheet、asset sheet、crop、切り出し。
- 1つの codex exec プロセスに複数画像のプロンプトをまとめて渡す（1プロセス=1画像厳守）。

## YAML

- `meta.layout_template`: `Scene01`〜`Scene21`
- `meta.pair`: `RM` / `ZM`
- `scenes[].scene_template`: 使用禁止
- 画像: `main.asset`
- asset: `assets/s01_main.png`
- subなし: `sub: null`
- 字幕: `dialogue[].text`
- `script_final.md` の自然な発話単位を維持
- 表示都合の機械分割は禁止
- `main.kind` は `image` のみ。sub枠ありテンプレートでは `sub.kind` は `text` / `bullets` のどちらか
- 新規動画で `sub.kind: image` は使わない
- `main.caption` / `main.text` / `main.items` / `sub.caption` は使わない。`sub` はテキスト・箇条書きを使う
- 字幕の折り返し・収まりはRemotion側で吸収する
- `bgm:` ブロックは必須
- `meta.width × meta.height` のデフォルトは FullHD（1920×1080）。レイアウトは 1920×1080 base から自動スケール
- 最終sceneは必ず `role: cta` にする
- title slot がないテンプレートでは `title_text` を書かない
- 各sceneに `motion_mode` を書く。値は `normal` / `punch` / `compare` / `warning` / `checklist` / `reveal` / `recap`
- 60秒以上 `normal` が続かないようにする
- 中盤再フック、ラスト行動は `normal` にしない
- s01、中盤再フック、ラスト行動には `dialogue[].emphasis` を最低1つ入れる
- `dialogue[].emphasis.words` には強調語、`style` には `punch` / `danger` / `surprise` / `number` / `action`、`se` には `pop` / `warning` / `question` / `reveal` / `success` / `fail` / `none`、`pause_after_ms` には `0` / `200` / `300` / `500` を使う

## 監査

PASSでも軽微な改善余地を1つ以上書く。

必ずFAIL:

- 5分で推定自然音声尺が下限未満。
- 5分で10シーン未満。
- 3分で60セリフ未満。
- 霊夢が質問だけ。
- ずんだもんが「そうなのだ」系ばかり。
- 魔理沙またはめたんの独演が多い。
- 章タイトルが説明目次。
- 画像プロンプトが抽象的。
- テンプレートのsub / subtitle / title / avoid_areaを考慮していない。
- YAML化で台本の面白さが消えている。
