# 01_台本生成プロンプト

## 目的

このプロンプトは、ゆっくり解説・ずんだもん解説動画の「台本」を生成するためのものです。

このファイルでは、**何を話すか・どんな順番で説明するか・会話として面白いか**だけを扱います。
編集演出、効果音、BGM、字幕フォント、画面エフェクト、Remotion/YMM4上の動きは指定しません。

---

## 重要な分離ルール

### 1動画1テンプレ固定

このプロジェクトでは、1本の動画につき `Scene01`〜`Scene21` のテンプレートを1つだけ使用します。
テンプレートは動画全体の見た目スキンであり、`scene` は時間ブロックです。
`script.yaml` では `meta.layout_template` に固定テンプレートを1つだけ指定し、各 scene は `main` / `sub` / `title_text` / `dialogue` の中身だけを変えてください。
`scenes[].scene_template` は使用禁止です。

以下はこのファイルでは扱わないでください。

- 効果音の指定
- BGMの指定
- 字幕フォントの指定
- テロップの縁取り指定
- 画像のズーム、フェード、シェイクなどの映像演出
- 章切り替えアニメーション
- キャラのバウンス、口パク、目パチなどの編集指定
- Remotion / YMM4 / AviUtl / DaVinci Resolve の実装指定

このファイルで出すのは、あくまで **台本・構成・会話・素材候補の内容メモ** までです。

---

## 入力情報

以下の情報を受け取って、動画台本を作成してください。

```text
動画タイプ：ゆっくり解説 / ずんだもん解説 / 自動選択
テーマ：{{THEME}}
想定尺：{{DURATION}}
想定視聴者：{{AUDIENCE}}
動画の目的：{{GOAL}}
トーン：{{TONE}}
必ず入れたい内容：{{MUST_INCLUDE}}
避けたい内容：{{AVOID}}
参考資料：{{REFERENCES}}
genre_profile：{{GENRE_PROFILE}}（任意。長尺ゆっくりの場合は genre_profiles/long-form-yukkuri.json の id）
layout_template：{{LAYOUT_TEMPLATE}}（Scene01〜Scene21。動画全体で1つだけ）
```

未入力の項目がある場合は、最も自然な前提で補完してください。
ただし、事実確認が必要な内容は推測で断定しないでください。
長尺ゆっくり台本では、必要に応じて `_reference/script_prompt_pack/genre_profiles/long-form-yukkuri.json` の `genre_profile` を選び、`script.yaml` の `meta.genre_profile` に id を記録してください。

## キャラクター構成

### ゆっくり解説の場合

- 霊夢：視聴者代表。疑問、驚き、ツッコミ、感情的リアクションを担当。
- 魔理沙：解説役。知識を整理して説明する担当。

基本比率：

- 魔理沙：60〜70%
- 霊夢：30〜40%

ただし、霊夢が質問し、魔理沙が説明するだけの単調な流れにしないでください。
霊夢にも仮説、誤解、反論、共感、ツッコミを入れて、会話として自然にしてください。

### ずんだもん解説の場合

- ずんだもん：視聴者代表。素直な疑問、ボケ、驚き、感情的リアクションを担当。
- 四国めたん：解説役。整理、補足、冷静なツッコミを担当。

基本比率：

- 四国めたん：55〜65%
- ずんだもん：35〜45%

ずんだもんは語尾に「なのだ」を使ってもよいですが、使いすぎて不自然にならないようにしてください。

## 台本の基本構成

以下の構成で作成してください。

```text
1. 冒頭フック
2. 挨拶
3. テーマ提示
4. 視聴者が気になる疑問の提示
5. お品書き・章立て
6. 第1章：前提・基礎知識
7. 第2章：核心・仕組み・理由
8. 第3章：具体例・比較・失敗例・意外な話
9. 第4章：結論・実用的な学び
10. まとめ
11. コメント誘導・次回予告
12. 締め
```

想定尺が短い場合は、章数を3章程度に圧縮してください。
想定尺が長い場合は、4〜6章まで増やしても構いません。

---

## 冒頭フックのルール

冒頭5〜10秒で、視聴者が続きを見たくなる一言を入れてください。

良いフックの例：

```text
霊夢：え、これって普通に使ってる人多いけど、実はかなり危ないってこと？
魔理沙：そうだぜ。知らないままだと、損するどころか普通に詰む可能性もあるんだ。
```

```text
ずんだもん：なんとなく便利そうって思ってたけど、そんな落とし穴があるのだ？
めたん：あるわ。しかも、多くの人が一番大事な部分を見落としているの。
```

冒頭で結論をすべて言い切らず、続きを見たくなる疑問を残してください。

---

## 面白い台本にするルール

単なる説明ではなく、以下を必ず入れてください。

- 視聴者が誤解しやすいポイント
- 「実は〇〇」という意外性
- 具体例
- 失敗例
- たとえ話
- キャラ同士の軽いツッコミ
- 最後に持ち帰れる結論

説明が長くなる場合は、霊夢・ずんだもん側に疑問やツッコミを挟ませてテンポを作ってください。

---

## 台詞の書き方

### 基本ルール

- 1セリフは短めにしてください。
- 1セリフで複数の論点を詰め込まないでください。
- 難しい用語は、初出時に会話内でかみ砕いて説明してください。
- 書き言葉ではなく、聞いて自然な話し言葉にしてください。
- 説明役だけが長く話し続けないようにしてください。

### 悪い例

```text
魔理沙：AI動画生成とは、生成AI技術を活用して画像、音声、字幕、構成要素を統合的に処理し、従来の動画編集工程を効率化する手法であり、近年ではSNSマーケティングや教育コンテンツ制作にも活用されています。
```

### 良い例

```text
魔理沙：AI動画生成っていうのは、ざっくり言うと、台本や画像や音声をAIに手伝わせて動画を作る方法だな。
霊夢：つまり、人間が全部手作業で編集しなくてもいいってこと？
魔理沙：そうだぜ。ただし、完全放置で神動画ができるわけじゃない。
```

## 素材候補メモの扱い

台本生成時点では、編集演出ではなく「どんな素材が必要そうか」だけをメモしてください。

例：

```text
素材候補：AIツールの画面イメージ、PC作業中のイラスト、比較表、注意マーク
```

以下のような編集指定はしないでください。

```text
NG：ここで画像をズームイン、SEドン、字幕を黄色にする
```

素材候補は、後続の `02_演出編集プロンプト.md` で具体的な演出に変換します。

## 品質チェック

出力後、以下を自己チェックしてください。

- 霊夢と魔理沙、またはずんだもんとめたんの役割が分かれているか
- 解説役だけが長く話しすぎていないか
- 冒頭に続きを見たくなるフックがあるか
- テーマを渡すだけでも成立する構成になっているか
- 章立てが自然か
- 難しい用語をかみ砕いているか
- 素材候補メモはあるが、編集演出まで踏み込んでいないか
- 最後に視聴者が得られる結論があるか

## 素材要件 (asset_requirements) 記述ルール

`script.yaml` の `main.asset_requirements` / `sub.asset_requirements` には、解決した **Image Engine** に応じてメタ情報を埋め込みます。
engine の解決手順とフォールバックチェーンはルートの `CLAUDE.md#Image Engine Workflow` を参照してください。

### engine = codex-imagegen の場合

各 `kind: image` の素材ごとに `imagegen_prompt` を必ず書きます。設計原則は `21_prompt_codex.md#2 Step 2.1` に準拠：

1. **構図** を最初に言う（中央に〜、左に〜）
2. **スタイル** を明示する（フラット / アイコン / 線画 / 3D / 水彩 ほか）
3. **色** を指定する（台本の image_style と揃える）
4. **アスペクト** を数値で書く（16:9 / 4:3 / 1:1）
5. **文字を含めない** を明記（画像内の文字は読めないゴミになる）
6. **negative** で弾きたい要素を書く

```yaml
main:
  kind: image
  asset: assets/s02_main.png
  caption: "DNA を切り貼りする分子ハサミ"
  asset_requirements:
    imagegen_prompt: |
      フラットデザインのイラスト、白背景。
      中央に DNA の二重らせんを分子ハサミ（はさみ型のアイコン）が切る構図。
      色は青緑系、線はくっきり。16:9 アスペクト比。
      テキスト・文字は一切入れない。
    style: "フラット・アイコン系"
    aspect: "16:9"
    negative: "写真風、リアル調、文字、ロゴ、人物の顔"
    nlm_marker_id: "[FIG:2]"   # フォールバック用、任意
```

### engine = notebooklm の場合（フォールバック）

NotebookLM の marker 形式（`[FIG:N]`、`[INFO:N]`、`[SLIDE:N]` 等）を使います。詳細は `notebookLM/templates/asset-marker-spec.md`。

```yaml
main:
  kind: image
  asset: assets/s02_main.png
  caption: "DNA を切り貼りする分子ハサミ"
  asset_requirements:
    nlm_marker_id: "[FIG:2]"
    description: "DNA二重らせんと分子ハサミの全体像"
    style: "フラット・アイコン系"
    aspect: "16:9"
```

### engine = text-fallback の場合

画像素材を要求せず、`main.kind: text` または `bullets` に縮退します。`asset_requirements` は不要です。
