# 台本監査・改善プロンプト

以下の台本を、YouTube解説動画として本当に面白く、視聴維持されやすい内容になっているか監査してください。
対象は「ゆっくり解説」または「ずんだもん解説」です。

## 監査対象
```text
ここに生成済み台本を貼る
```

## 監査観点

### 1. キャラ構成
- ゆっくり解説の場合、霊夢が視聴者代表、魔理沙が解説役になっているか
- ずんだもん解説の場合、ずんだもんが疑問・感情役、めたんが解説・整理役になっているか
- 解説役が長文独演していないか
- リアクション役が相槌だけになっていないか
- 語尾や口調が崩れていないか

### 2. テーマだけで作れるか
- テーマの前提をAIが補完できているか
- 想定視聴者が明確か
- 結論が1文で言えるか
- 章立てが自然か
- 不明情報を捏造していないか

### 3. バズる可能性
- 冒頭5秒に意外性・損得・誤解訂正があるか
- タイトルとサムネがクリックしたくなるか
- 中盤に離脱防止の転換があるか
- 視聴者がコメントしたくなる問いがあるか
- 最後に人に話したくなる一言があるか

### 4. 分かりやすさ
- 専門用語が噛み砕かれているか
- 1セリフ1目的になっているか
- たとえ話が分かりやすいか
- 情報が詰まりすぎていないか
- 同じ内容の繰り返しがないか

### 5. 動画化しやすさ
- 画面枠メモがあるか
- 強調候補メモがあるか
- 必要素材リストが具体的か
- 検索キーワードが素材探しに使えるか
- BGM・SE・確定演出を台本段階で決めすぎていないか
- 最終演出は `02_演出編集プロンプト.md` に渡す前提になっているか

### 5-1. Image Engine 整合性
- 解決した engine（`codex-imagegen` / `notebooklm` / `text-fallback`）が台本冒頭または `script.yaml.meta` に記録されているか
- engine = codex-imagegen の場合、各 `kind: image` に `asset_requirements.imagegen_prompt` が書かれているか
- `imagegen_prompt` が「構図 → スタイル → 色 → アスペクト → 文字なし → negative」の順で書かれているか
- 「テキスト・文字は一切入れない」が明記されているか
- `aspect` が数値（16:9 / 4:3 / 1:1）で書かれているか
- engine = notebooklm の場合、`asset_requirements.nlm_marker_id` または台本中の `[FIG:N]` 等のマーカーが具体的か
- engine = text-fallback の場合、画像素材を要求せず `main.kind: text` / `bullets` に縮退しているか

### 6. 意味的チェック（実例パターン検出）

形式チェックだけでなく、台本が実機で機能するかを実例パターンで判定する。
詳細チェック ID は `_reference/script_prompt_pack/_quality-checklist.md` を参照。

#### 6-1. 単調サイクル検出（Q-01）
- 「短い疑問（霊夢／ずんだもん）→ 短い説明（魔理沙／めたん）」だけのサイクルが **3 章以上連続していないか**
- 検出方法：1 往復が 3〜4 セリフで完結し、次の章でも同じパターンが続いていれば NG
- 失敗例：`script/ep501-rm-body-trivia/script.yaml:80-92, 145-163` の身長／椎間板パターンが 20 章連続

#### 6-2. 意外性ファクトの存在（Q-03）
- 各章に「実は」「意外にも」「ところが」「驚くべきことに」型の意外性ファクトが最低 1 つあるか
- 検出方法：章単位で grep。各章で 1 つ以上ヒットしないなら NG

#### 6-3. 中盤フックの実機チェック（S-03）
- 台本の 40〜60% 地点に「ここまで聞くと〜だけど、実は〜」型または「でも本当にヤバいのは〜」型の転換セリフが存在するか
- 単に「中盤に何かある」ではなく、視聴離脱を実際に防ぐ強さの転換があるかで評価

#### 6-4. 締めの「話したくなる一言」（S-05）
- エンディングに、視聴後に視聴者が誰かに話したくなる一文があるか
- OK 例：「たった 1 種類残った人類、奇跡のように与えられた命を私たちは大切にしないといけないわね」
- NG 例：「全部覚えきれた？」「一つでも OK」型の無責任まとめ（`ep501-rm-body-trivia/script.yaml:739-757` 参照）

#### 6-5. キャラ口調崩れの具体検出（C-01〜03）
- 霊夢／ずんだもんの語尾に解説役の語尾（「だぜ」「のよ」等）が混入していないか
- 魔理沙／めたんの語尾に疑問役の語尾（「のね」「のだ」等）が混入していないか
- 検出方法：各セリフの語尾を抽出して照合

#### 6-6. 応酬の層（Q-02）
- 各章に「疑問 → 説明 → 反論 or 補足 → 新発見」の 4 層が最低 1 回あるか
- 単なる「疑問 → 説明」の 2 層止まりは深さ不足、NG

#### 6-7. アーキタイプ整合（A-01〜03）
- `_reference/script_prompt_pack/archetypes/{list-type, case-study-type, science-mystery-type}.md` の章構造に沿っているか
- 該当アーキタイプの「典型的な失敗パターン」表（§ 8 / § 9）に該当していないか
- アーキタイプ固有の必須要素を満たしているか（list: 橋渡し台詞・余韻クロージング、case-study: 推定無罪パート、science-mystery: 冒頭フック回収）
