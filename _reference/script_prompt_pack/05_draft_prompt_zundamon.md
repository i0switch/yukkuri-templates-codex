# 05_draft_prompt_zundamon

## 目的

ZMペア、ずんだもん・四国めたんの高品質な解説台本を作る。

## キャラ

- ずんだもん: 視聴者代表。賢くしすぎない。勘違い、ボケ、雑な理解、焦り、ツッコミを担当。
- 四国めたん: 解説役。冷静に刺す。短く訂正し、具体例、数字、結論を出す。

## 出力形式

```md
# 台本ドラフト ZM

## メタ
- episode_id:
- layout_template:
- pair: ZM
- target_duration:
- target_dialogue_count:

## s01: フック型タイトル
- role:
- scene_format:
- scene_goal:
- viewer_question:
- viewer_misunderstanding:
- reaction_level:
- number_or_example:
- main_content:
- sub_content:  # sub 枠ありテンプレ Scene02/03/10/13/14 では空欄禁止。textまたはbullets用の補足を必ず書く
- image_insert_point:
- mini_punchline:

ずんだもん「」
めたん「」

visual_asset_plan:
  - slot: main
    purpose:
    adoption_reason:
```

`hook_type` / `boke_or_reaction` / `visual_type` / `composition_type` / `supports_dialogue` / `supports_moment` / `subtitle_area` / `title_area` は draft メタに置かない。v2 では使用しない（テンプレートのレイアウトで吸収）。

## 必須

- 冒頭の最初の1〜3発話は日常前提から入る。
- 「強いパスワードを使い回せば楽」のような、前提なしの雑な解決策始まりは禁止。
- 冒頭5秒で損失、衝撃数字、誤解訂正、強い疑問、あるあるからの意外性のいずれかを刺す。
- 冒頭1番目のセリフは次の5タイプから最低1つを採用する。
  - 損失提示: 「無料のつもりが、毎月980円抜かれてたのだ」
  - 衝撃数字: 「その充電、バッテリー寿命を2年縮めてるかも」
  - 誤解訂正: 「サブスク解約、アプリ消すだけじゃ止まらないのだ」
  - 強い疑問: 「なんで使ってないアプリに毎月払ってるのだ？」
  - あるある→意外: 「節約あるある。でも一番漏れてるの、買い物じゃなかったのだ」
- 「あるある」だけ、「今回は〜について」、「こんにちは」、「〇〇って知ってる？」だけの冒頭はFAIL。
- 日常の状況、視聴者がやりがちな行動、なぜ今この話をするかを置いてから、損失や危険性へ接続する。
- ずんだもんを質問係だけにしない。
- 1シーン内のずんだもん発話のうち、疑問形（？で終わる発話）は最大1個。それ以外は、ボケ、誤解、強い反応、ツッコミ、行動宣言、実体験のいずれかにする。
- 各本編シーンのずんだもんの1発目は、単なる質問ではなく、勘違い、言い訳、視聴者の本音、雑な解決策、焦り、疑い、的外れなたとえ、自分に都合のいい解釈のどれかにする。
- ずんだもんの「なのだ」「のだ」は自然に混ぜる。全発話には付けない。
- 「だね」「かな」「じゃん」「マジで？」なども混ぜる。
- めたんは説明に入る前に、8〜18字程度の短いツッコミまたは訂正を入れる。
- めたんは2セリフ以内でずんだもんに返す。
- めたんだけが3セリフ以上続かない。
- L3以上リアクションを最低2回。
- 各シーンに数字、具体例、失敗例、あるある、比較のいずれかを入れる。
- 中盤シーン（全体の40〜60%地点）には、衝撃数字の再提示、失敗例の生々しい描写、視聴者煽り、比較のいずれかを最低1つ入れる。
- 最後から2番目のシーンには、投票形式の問いかけ、共感誘発のあるある、強めの言い切りのいずれかを入れる。
- 最後は具体行動で終える。
- `scene_format`、`viewer_misunderstanding`、`reaction_level`、`mini_punchline`、`セルフ監査` という品質マーカーを含める。
- sub枠ありテンプレートでは全シーンの `sub_content` を必須にする。詳しい補足、注意点、小見出し、4〜6項目チェック、NG/OK、手順現在地、次にやることのいずれかを書く。
- `sub.kind: bullets` に変換できる内容は原則4〜6項目、推奨6項目ぶん書く。
- `sub.kind: text` に変換する内容は4〜6行の短い補足文として書く。
- 3項目だけに絞るのは、締め・要約・最終行動など、情報を絞る意図が明確なシーンだけにする。

## セリフ

- 表示都合で先に切り詰めない。
- 長い説明は、意味や掛け合いの切れ目で自然に言い換える。
- ずんだもん40、めたん60目安。
- 「なのだ」「のだ」は20〜40%目安。
