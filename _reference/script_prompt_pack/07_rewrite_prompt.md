# 07_rewrite_prompt

## 目的

監査FAIL箇所だけを直す。台本全体をむやみに作り直さない。

## 優先順位

1. 尺不足、シーン不足、セリフ不足。
2. 聞き役化、Q&A化、条件なしの抽象独演。
3. 章タイトルの説明目次化。
4. 数字、具体例、あるある不足。
5. リアクション強度不足。
6. 口癖の偏り。
7. テンプレート枠との不整合。
8. 画像プロンプトとの不整合。
9. 最終行動の弱さ。

## ルール

- 修正に入る前に、必ず原因分類を行う。
- 問題のないシーンは維持する。
- 尺不足時は不足分だけを台本補完する。自然音声尺が allowed_max を超えているだけの場合は、そのまま使う。
- `recommended_action: keep_natural_overrun_do_not_trim`、`duration_status: over`、または allowed_max 超過 warning は、修正依頼ではなく保持指示として扱う。尺オーバー対策として `script_final.md` を削除、短文化、圧縮してはいけない。
- `script_final.md` を補完した後は `audits/script_final_review.md` が stale になるため、必ず再レビューしてからYAML化へ戻る。
- 解説を増やす場合も、掛け合いの温度差を残す。
- 解説役の条件なし独演は視聴者代表の反応で分割する。根拠説明、手順説明、注意点整理、比較説明は最大4セリフまで許可するが、説明前後に視聴者代表の誤解や反応を置き、具体例・数字・比較・手順・失敗例のうち最低2つを入れる。
- 画像プロンプトを直す場合は、先に `script_final.md` の対象シーン本文と作風キーワード（数語）を直す。中間ファイル `image_direction` には依存しない。
- 抽象プロンプトへ戻さない。`hook_type` / `visual_type` / `composition_type` / `supports_*` を `imagegen_prompt` 本文に混ぜない。

## 修正前の原因分類

各修正対象に、必ず次のいずれかの原因分類を付ける。

- `PROMPT_CAUSE`: 生成プロンプトの指示不足。
- `AUDIT_CAUSE`: 監査が見逃した。
- `STRUCTURE_CAUSE`: 構成設計が弱い。
- `CHARACTER_CAUSE`: キャラ役割が崩れた。
- `FACT_CAUSE`: 数字・根拠・出典が弱い。
- `TEMPLATE_CAUSE`: レイアウトやsub枠との接続が弱い。

例:

| 修正対象 | 問題 | 原因分類 | 修正方針 |
|---|---|---|---|
| s01 | 同じ説明が2回出ている | AUDIT_CAUSE | 重複検出ルールを追加し、片方を生活場面へ置き換える |

FAILが `AUDIT_CAUSE` または `PROMPT_CAUSE` を含む場合は、`audits/prompt_failure_feedback.md` に次回防止ルールを残す。

## 参考

修正後、`00_MASTER_SCRIPT_RULES.md` と対象ペアの draft prompt の最終自己確認に照らして、同じ問題が残っていないことを確認する。`examples/` 配下は人間監査や追加学習用の補助資料であり、通常生成の必読入力ではない。

```md
# 台本修正版

## 1. 修正概要
| 修正対象 | 問題 | 原因分類 | 修正方針 |
|---|---|---|---|

## 2. 修正後の該当箇所

### sXX: フック型タイトル
修正後のシーン全文。

## 3. 修正後確認
- Blocking Issues 解消:
- 尺と密度:
- キャラ口調:
- 条件なしの抽象独演なし:
- テンプレート適合:
- 画像プロンプト適合:
- 軽微な改善余地:
- 判定:
```
