# 04_rewrite_prompt

## 目的

監査結果をもとに、問題箇所だけを修正する。
台本全体をむやみに再生成しない。
必ず `00_MASTER_SCRIPT_RULES.md`、対象台本、`03_audit_prompt.md` の監査結果を読む。

## 入力

```text
対象台本：
監査結果：
修正対象：
```

## やること

- Blocking Issues を最優先で直す。
- 想定尺に対する本文量不足は最優先で直す。
- Important Issues を必要最小限の差分で直す。
- 問題のないシーンは維持する。
- 情報量を増やすより、掛け合いの温度差を優先する。
- 既存の正しい説明は維持し、会話の入口をボケ・誤解・あるあるに変える。
- 章タイトルをフック型に直す。
- 数字・具体例・あるあるを不足シーンに追加する。
- ずんだもん、霊夢などの視聴者代表キャラを質問係で終わらせない。
- めたん、魔理沙の独演を分割する。
- L3以上のリアクションが不足していれば、驚き、疑い、焦り、呆れを分散して追加する。
- 3から5シーンに1回、短い例え話・小ネタを入れる。
- ZMでは「なのだ」「のだ」率を20〜40%目安に調整する。50%以上は過剰、10%未満はキャラ性不足として直す。
- 同じ語尾を3連続させない。
- 強すぎる煽りや根拠のない数字は使わない。
- 実在ブランドUIの模写を避け、必要に応じて一般化する。
- キャラ口調、テンプレート枠、素材計画、参照動画型メタを壊さない。
- `visual_asset_plan` と `imagegen_prompt` は維持し、必要な場合だけシーン内容に合わせて更新する。
- `imagegen_prompt` を更新する場合、【用途】【主題】【構図】【テンプレート枠】【色】【情報量】【絵柄】【禁止】の8項目を維持する。
- 抽象化して「フラットな図解」「わかりやすいカード」「中央に主題、余白多め」だけへ戻さない。
- `imagegen_prompt` には scene_id、slot、字幕帯/キャラ位置回避、画像内文字0文字または日本語3〜6文字以内の指示を残す。
- main枠とsub枠の役割分担を崩さない。sub枠に複雑な図や長文を追加しない。
- 5分前後なら10〜12シーン、90〜130セリフ、平均8セリフ以上を目安に増補する。
- 各シーンが6セリフ固定の場合、深掘り往復を追加して8〜12セリフにする。
- 中盤40〜60%地点に再フックを追加する。
- 修正後にジャンル適合スコアと尺・密度チェックを再計算する。
- 修正後にセルフチェックを行い、PASS / FAIL を出す。
- 最終行動が「見るだけ」で終わっている場合、確認、1つ選ぶ、保存/逃がす/テンプレ化、予定化など2アクション以上へ強める。
- 章タイトルにテーマ外の単語が混ざっている場合、テーマ語彙に戻して修正する。
- 助詞抜け、誤字っぽい語尾、不自然な口語を必ず修正する。
- RMでは魔理沙の「だぜ」率を30〜60%目安に調整し、過剰なら一部外す。
- 修正後も軽微な改善余地を1つ探し、セルフ監査を甘くしない。

## リライト優先順位

1. 尺不足、シーン不足、セリフ不足を直す。
2. 聞き役化、Q&A化、独演などの会話構造を直す。
3. 章タイトルをフック型へ直す。
4. 不足シーンへ数字・具体例・あるあるを足す。
5. リアクション強度と口癖の偏りを直す。
6. scene_format の不足を直す。
7. visual_asset_plan / imagegen_prompt との整合を取る。
8. 最終行動の弱さ、テーマ外タイトル、自然文の違和感を直す。
