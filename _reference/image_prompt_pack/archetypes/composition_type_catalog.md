# Composition Type Catalog

ゆっくり解説 / ずんだもん解説動画における 15 種の composition_type 定義カタログ。
`script.yaml` の `asset_requirements.composition_type` で参照する。visual_type が「何を見せるか」を決め、composition_type は「どう配置するか」を決める。`imagegen_prompt` の構図段で本ファイルの「前景 / 中景 / 背景 / 視線誘導 / 余白」を引き写すこと。

## smartphone_closeup

- **構図**: 画面の左半分または右半分にスマホをクローズアップ配置。残り半分に対比要素（STOP / 安全 / 比較カード）。
- **前景**: 押しそうな指 or 警告サイン
- **中景**: スマホ画面（抽象通知 / カード）
- **背景**: 淡いグレー or 注意喚起ポスター風
- **視線誘導**: 前景の指先 → スマホ画面 → 対比要素
- **Remotion overlay 余白**: 上部左 36% / 下部 20%
- **相性の良い visual_type**: hook_poster / danger_simulation / boke_visual
- **避けるべきこと**: スマホ全画面化、字幕帯にスマホ下部が侵入

## diagonal_flow

- **構図**: 斜めの矢印で危険導線を見せる、左上 → 右下。3〜4 段の流れを斜め配置で勢いを付ける。
- **前景**: 太い斜め矢印（オレンジ → 赤のグラデーション）
- **中景**: 各段のアイコン（タップ → 偽サイト → 入力 → 流出）が斜め線上に配置
- **背景**: 薄グレー＋薄い格子模様で動きを抑える
- **視線誘導**: 左上の出発点 → 矢印に沿って右下の最終結果へ自然に視線が下がる
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。矢印終点は字幕帯の上で止める
- **相性の良い visual_type**: danger_simulation / flowchart_scene / mini_story_scene
- **避けるべきこと**: 矢印が逆向き（右下 → 左上）、矢印が画面端で切れて結末が見えない

## product_shot

- **構図**: 商品広告風。中央に主役オブジェクト＋小道具を配置し、商品カタログのような完成度に寄せる。
- **前景**: 主役オブジェクトの手前に小道具（メモ、コーヒー、メガネ）を散らす
- **中景**: 主役オブジェクト（スマホ / アプリアイコン / セキュリティキー）
- **背景**: 単色の薄いグラデーション、影は柔らかく
- **視線誘導**: 周辺小道具 → 中央主役へ収束
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。主役オブジェクトは中央 50% 帯に収める
- **相性の良い visual_type**: ui_mockup_safe / hook_poster / final_action_card
- **避けるべきこと**: 主役が小さすぎて何が商品か読めない、背景が騒がしくて主役が埋もれる

## three_cards

- **構図**: 横 3 枚カード、final_action_card 御用達。各カードは同サイズ、間隔は均等、矢印で連結。
- **前景**: カード番号アイコン（①②③）と矢印
- **中景**: 各カード内のアクション動詞＋代表アイコン
- **背景**: 薄グレー単色＋微細なドットパターン
- **視線誘導**: 左カード → 矢印 → 中央カード → 矢印 → 右カード
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。3 カードは中央 60% 帯に収める
- **相性の良い visual_type**: final_action_card / three_step_board / mini_story_scene
- **避けるべきこと**: カードサイズが揃わない、矢印が省略される、4 枚目が混入する

## split_danger_safe

- **構図**: 左右で危険 / 安全を対比、中央に分岐線。左 = 危険、右 = 安全（読み進める方向と一致）。
- **前景**: 中央の太い分岐線（または `\` 形の岐路）
- **中景**: 左に危険アイコン、右に安全アイコン
- **背景**: 左半分は薄ピンク〜赤、右半分は薄青緑
- **視線誘導**: 中央分岐線 → 左右どちらの世界に行くかを比較する視線運動
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。分岐線が字幕帯にかからないこと
- **相性の良い visual_type**: tsukkomi_visual / myth_vs_fact / danger_simulation
- **避けるべきこと**: 危険側を右に置く（読み進める方向と逆になる）、中央線がなく対比が曖昧

## security_visual

- **構図**: 盾アイコン＋鍵＋抽象データ。中央に盾、その周囲にデータ流＋鍵を配置。
- **前景**: 盾アイコン（青緑、太い枠線）
- **中景**: 鍵アイコン、抽象データブロック（点線で囲まれた箱）
- **背景**: 暗めの紺〜青グラデーション、格子状のセキュリティパターン
- **視線誘導**: 周辺データ → 中央盾 → 鍵で「守られている」結論
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。盾は中央 50% 帯に収める
- **相性の良い visual_type**: tsukkomi_visual / final_action_card / contrast_card
- **避けるべきこと**: 実在ロゴ・実在ブランドの盾形を使う、写真調にする、背景が騒がしくて盾が埋もれる

## consultation_flow

- **構図**: 相談者 → 助言者 → 解決の 3 段構成。横並びまたは S 字配置。
- **前景**: 各キャラの簡略シルエット（顔は描かない）と吹き出し
- **中景**: 中央に矢印または会話線で 3 段を結ぶ
- **背景**: 薄黄〜薄青緑、相談カウンター風
- **視線誘導**: 左の相談者 → 中央の助言者 → 右の解決アイコンへ流れる
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。3 段は中央 60% 帯に収める
- **相性の良い visual_type**: mini_story_scene / before_after / three_step_board
- **避けるべきこと**: 助言者がキャラ立ちしすぎて霊夢 / ずんだもんと衝突、会話線がなく順序が不明

## before_after_split

- **構図**: 左 before / 右 after、中央に矢印。左右のカードサイズは完全に揃え、中央矢印で時間進行を示す。
- **前景**: 中央の太い右向き矢印（黄色）と「Before」「After」ラベル
- **中景**: 左カード = 乱雑な状態、右カード = 整った状態
- **背景**: 左 = 薄グレー、右 = 薄青緑
- **視線誘導**: 左 Before → 中央矢印 → 右 After へ自然に流れる
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。中央矢印は字幕帯にかけない
- **相性の良い visual_type**: before_after / contrast_card / three_step_board
- **避けるべきこと**: After を左に置く、中央矢印がなく時間進行が読めない、左右サイズがズレる

## ranking_podium

- **構図**: 表彰台型。中央 1 位を最も高く、左 2 位、右 3 位。または縦並びカードで上から 1 位。
- **前景**: 各順位ラベル（金 / 銀 / 銅）と項目アイコン
- **中景**: 表彰台のブロック（高さで順位を示す）
- **背景**: 薄グレー＋スポットライト風の放射グラデ
- **視線誘導**: 中央 1 位 → 左 2 位 → 右 3 位、または上から順にスクロール
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。表彰台は中央 60% 帯に収める
- **相性の良い visual_type**: ranking_board / contrast_card / final_action_card
- **避けるべきこと**: 1 位が中央以外に配置される、順位が同じ高さで描かれる、4 位以下を入れる

## flow_chart_horizontal

- **構図**: 横向きフロー、4〜5 段。左から右へ読み進める。
- **前景**: 各ノードのアイコン＋短いラベル（4〜6 文字）
- **中景**: ノード間の矢印（細めだが視認できる太さ）
- **背景**: 薄グレー＋微細グリッドで方向感を出す
- **視線誘導**: 左端の起点 → 右端の終点まで一直線
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。フロー全体は中央 60% 帯に収める
- **相性の良い visual_type**: flowchart_scene / danger_simulation / three_step_board
- **避けるべきこと**: 6 段以上に伸ばす、矢印が交差する、ノードサイズがバラバラ

## flow_chart_vertical

- **構図**: 縦向きフロー、3〜4 段。上から下へ読み進める。サブ枠やスマホ視聴向けに縦長で収める。
- **前景**: 各ノードのアイコン＋短いラベル
- **中景**: ノード間の下向き矢印
- **背景**: 薄グレー＋縦のガイドライン
- **視線誘導**: 上端の起点 → 下端の終点まで縦に流れる
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。フロー全体は中央 50〜60% 帯に縦長で収める
- **相性の良い visual_type**: flowchart_scene / three_step_board / checklist_panel
- **避けるべきこと**: 5 段以上に伸ばす、矢印が右や左に逸れる、字幕帯まで縦に伸ばす

## isometric_scene

- **構図**: アイソメトリック視点（30 度俯瞰）、机 / 部屋 / オフィス。立体感のあるイラストで全体像を見せる。
- **前景**: 机上の小道具（ノート、スマホ、コーヒー）と中心オブジェクト
- **中景**: 机そのもの、椅子、壁面の小物
- **背景**: 部屋の輪郭は薄く、フロアパターンで奥行きを出す
- **視線誘導**: 机中央のメインオブジェクト → 周辺小道具 → 部屋全体へ視線が広がる
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。中心オブジェクトは画面中央 50% 帯に置く
- **相性の良い visual_type**: mini_story_scene / before_after / hook_poster
- **避けるべきこと**: 視点角度がバラバラ（一部だけ正面構図）、机の縁が画面端で切れて構図が崩れる

## desk_top_down

- **構図**: 机を真上から俯瞰、ノート / スマホ / コーヒー。フラットレイ写真の構図をイラスト化する。
- **前景**: なし（俯瞰なので「前」がない）
- **中景**: 机上のオブジェクト群（ノート、スマホ、ペン、コーヒー、メモ）
- **背景**: 木目またはコルク調の机表面
- **視線誘導**: 中央のメインオブジェクト → 周辺の小道具へ視線が広がる
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。机は画面いっぱいに敷くが、字幕帯に重なる小道具は薄く描く
- **相性の良い visual_type**: checklist_panel / mini_story_scene / contrast_card
- **避けるべきこと**: 視点が斜めに崩れる（真上ではなくなる）、小道具が多すぎて主役が埋もれる

## character_silhouette_avoid

- **構図**: キャラ立ちのテンプレで、左 30% / 右 30% を避け中央 40% にメイン要素。霊夢 / 魔理沙 / ずんだもん / めたんが画面に重なるレイアウト用。
- **前景**: 中央 40% 帯に主役オブジェクトを大きく配置
- **中景**: 中央帯の周辺に補助要素（矢印、小ラベル）
- **背景**: 薄グレー単色 or 微細パターン。両端 30% は意図的にスカスカ
- **視線誘導**: 中央 40% に視線が集中するよう、左右はトーンを落とす
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕、左右 30% はキャラ重畳前提で要素を置かない
- **相性の良い visual_type**: hook_poster / boke_visual / tsukkomi_visual / final_action_card
- **避けるべきこと**: 左右 30% に重要要素を入れる（キャラに隠れる）、中央が逆に空っぽ、両端まで密に描き込む

## mid_hook_zoom

- **構図**: 中盤再フック用、ズームイン感のあるカット。何かを「拡大して見せる」演出。
- **前景**: 拡大対象（虫眼鏡の中、トリミング枠、点線拡大線）
- **中景**: 元の全景の一部が左右に薄く見える
- **背景**: 周辺はぼかし、拡大対象だけクッキリ
- **視線誘導**: 周辺のぼけ → 中央拡大対象へ強制的に視線を引き込む
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。拡大対象は中央 50% 帯に収める
- **相性の良い visual_type**: hook_poster / danger_simulation / myth_vs_fact
- **避けるべきこと**: 拡大対象が小さくて何を見せたいか不明、周辺がクッキリしすぎて中央が埋もれる、虫眼鏡が小道具的に飾られて拡大効果がない
