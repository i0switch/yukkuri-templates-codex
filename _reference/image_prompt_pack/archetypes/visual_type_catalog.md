# Visual Type Catalog

ゆっくり解説 / ずんだもん解説動画における 15 種の visual_type 定義カタログ。
`script.yaml` の `asset_requirements.visual_type` で参照する。台本フェーズで visual_type を選んだら、本ファイルから役割・構図ヒント・配色・禁止パターンを引き写し、`imagegen_prompt` 設計に反映すること。

## hook_poster

- **役割**: 冒頭で「何がヤバいか」「なぜ見るべきか」を一発で見せる。動画オープニングや s01 で使う最重要型。
- **会話パターン**: 強い損失型 / 危険喚起 / 意外性 / 視聴者代表が驚いた直後
- **典型シーン**: s01 / 動画内 mid-hook の再起点
- **構図ヒント**: 危険を示す主役（赤い STOP / 赤ゲージ / 警告マーク）と、視聴者がやりがちな行動（指、入力、保存）が同居。注意喚起広告ポスターの完成度に寄せる。
- **配色**: 危険箇所だけ赤、ベースは青緑＋薄グレー
- **ep904（詐欺DM）例**: 当選通知＋STOP サイン＋押しそうな指
- **ep001（認知バイアス）例**: ぐるぐる回る矢印＋脳のシルエット＋「即決」の短語
- **禁止パターン**:
  - 単独のシンプルアイコンだけ
  - 「STOP」のみ大きく書いた標識画像
  - キャラの顔メイン
- **Remotion overlay 余白**: 上部左に 60×16 grid 程度のタイトル空き、下部 20% は字幕＋キャラ

## boke_visual

- **役割**: 視聴者代表（霊夢 / ずんだもん）の素朴な誤解・楽観・思い込みを画面で見える化する。ボケの直後、ツッコミの前に置く。
- **会話パターン**: 軽い楽観 / 「なんとかなる」誤解 / 「無料なら安全」「これくらい大丈夫」
- **典型シーン**: s02〜s03 / 中盤再フック直前のミスリード
- **構図ヒント**: ボケ役のセリフを示す吹き出し的要素＋安易な行動アイコン（OKマーク、ハート、緑チェック）。意図的にゆるく見せ、ツッコミ visual と対比できる構図にする。
- **配色**: パステル中心、警告色は使わずにあえて呑気さを出す
- **ep904（詐欺DM）例**: 「当選おめでとう」吹き出し＋にっこり顔のスマホ＋緑チェック
- **ep001（写真3枚消し）例**: 「写真3枚消せば勝ち」短語＋手のひらに3枚の写真アイコン
- **禁止パターン**:
  - 警告色（赤）を多用する
  - 危険な結末を同じ画に同居させる
  - キャラの顔をリアル描写する
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 22% は字幕＋ボケキャラ

## tsukkomi_visual

- **役割**: 解説役のツッコミ・正解導入を視覚化する。NG ルートと正解ルートの違いを「順序」「配置」「色」で訴える。
- **会話パターン**: 「それが危ない」 /「順番が逆」 /「やるなら先にこっち」
- **典型シーン**: ボケ visual の直後 / 中盤の方針提示
- **構図ヒント**: NG 側を赤マーク、正解側を青緑マーク。中央に分岐線または矢印反転。洗練された画面で、ボケ visual と地続きに見えるレイアウトにする。
- **配色**: NG = 赤＋濃グレー、正解 = 青緑＋白、背景は薄グレー
- **ep904（詐欺DM）例**: 「URL を踏む」赤バツ ↔「公式アプリで確認」青緑チェック
- **ep001（写真3枚消し）例**: 「写真 3 枚」赤グレー ↔「アプリ内データ＋クラウド整理」青緑
- **禁止パターン**:
  - NG と正解の色を逆にする
  - 文字でしか説明していない
  - キャラの顔だけで結論を言わせる
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 22% は字幕＋ツッコミキャラ

## myth_vs_fact

- **役割**: 視聴者の典型的な誤解を、左に提示し右で訂正する。
- **会話パターン**: 「〜だと思ってた」 /「実はこう」 /「クラウドあるから安心」→「同期未完了は消える」
- **典型シーン**: 中盤の認識訂正パート / mid-hook 後の再導入
- **構図ヒント**: 左：誤解（吹き出し＋誤った行動）/ 中央：太い矢印または分岐記号 /右：実際（正しい挙動と数値）。左右の高さを揃え、対比を一目で読ませる。
- **配色**: 左 = 薄ピンク〜グレー、矢印 = 黄、右 = 青緑＋白
- **ep904（詐欺DM）例**: 左「公式に見える DM だから本物」 → 右「公式は DM 当選通知を送らない」
- **ep001（クラウド誤解）例**: 左「クラウドあるから消えても大丈夫」 → 右「同期未完了は端末側だけで消える」
- **禁止パターン**:
  - 左右の重要度が偏ってどちらかが小さい
  - 中央の矢印がない / 何の比較かわからない
  - 文字だけで対比している
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕

## danger_simulation

- **役割**: 危険進行のシミュレーション（A → B → C と被害が広がる流れ）を見せ、視聴者を「この経路に乗ったら戻れない」と納得させる。
- **会話パターン**: 強い因果列挙 / 「押すと → 入力すると → 抜き取られる」
- **典型シーン**: 危険プロセスを 1 シーンで端的に伝える場面
- **構図ヒント**: 3〜4 段の連続フローを 1 枚に収める。各段に短いラベル（DM / 偽サイト / フォーム入力 / 流出）。最終段の被害は赤で強調。
- **配色**: 段が進むほど赤味が増す。最終段だけ濃い赤、中継段は黄色〜オレンジ
- **ep904（詐欺DM）例**: DM タップ → 偽ログイン画面 → ID/Pass 入力 → アカウント乗っ取り
- **ep001（容量不足）例**: 容量赤ゲージ → 撮影失敗 → バックアップ停止 → 同期不能
- **禁止パターン**:
  - 段がバラバラに散在して順番が読めない
  - 最終被害が描かれずぼやけている
  - 1 段だけで終わっている（フローではない）
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。フローは中央 60% 帯に収める

## before_after

- **役割**: 古いやり方／間違ったやり方と、新しいやり方／正しいやり方を視覚的に対比する。改善動機の一押しを担う。
- **会話パターン**: 「以前はこうだった」 / 「今はこうする」 / 改善提案直後
- **典型シーン**: 解決策提示時 / 動画後半の整理パート
- **構図ヒント**: 左右分割。Before 側は乱雑・赤系・劣化感。After 側は整然・青緑系・余白多め。中央には矢印または改善ラベル。
- **配色**: Before = グレー＋赤味、After = 青緑＋白。中央矢印は黄
- **ep904（詐欺DM）例**: Before「DM 内 URL を即タップ」 / After「公式アプリで通知確認」
- **ep001（容量整理）例**: Before「ホーム画面ぐちゃぐちゃ＋赤ゲージ」 / After「整理されたフォルダ＋緑ゲージ」
- **禁止パターン**:
  - Before と After のレイアウトが違いすぎて比較にならない
  - どちらが After か読めない（色・矢印で示せていない）
  - 中央の矢印・ラベルがなく分断している
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。中央矢印は字幕帯にかけない

## three_step_board

- **役割**: 視聴者が今すぐできる 3 ステップを 1 枚で提示する。最終行動カードと並ぶアクション提示型。
- **会話パターン**: 「やることは 3 つ」 / 「①止まる ②確認 ③ブロック」
- **典型シーン**: 解決策パート / 動画後半の指針提示
- **構図ヒント**: 横 3 段または縦 3 段。各段に番号アイコン＋短いラベル（4 文字以内推奨）＋小さな代表アイコン。段間は矢印で結ぶ。
- **配色**: 全段同じ青緑系で統一。番号アイコンだけアクセントカラー（橙）
- **ep904（詐欺DM）例**: ①止まる ②公式で確認 ③ブロック / 通報
- **ep001（写真整理）例**: ①確認 ②退避 ③削除
- **禁止パターン**:
  - 段が 4 つ以上ある（3 ステップ型ではなく flowchart になる）
  - 番号がなくて順序が読めない
  - 各段の文字量が多くて読み切れない
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。各段は中央 60% 帯に収める

## checklist_panel

- **役割**: サブ枠用の 3 項目チェック。NG ワードリスト・行動リスト・確認項目をコンパクトに提示する。
- **会話パターン**: 「次の 3 つを覚えて」 / 「これが入っていたら危険」
- **典型シーン**: サブ枠（`sub.kind: image`）/ 動画後半の判断軸提示
- **構図ヒント**: 縦 3 行のチェックリスト。各行はチェックボックス＋短いラベル（10 文字以内）。背景はメモ用紙風 or カード風。
- **配色**: 白〜薄青緑カード＋濃いグレー文字。チェックは青緑
- **ep904（詐欺DM）例**: ☑ 当選 / プレゼント / ☑ URL 付き / ☑ 即時行動を促す
- **ep001（容量整理）例**: ☑ アプリ内データ / ☑ 重複写真 / ☑ 動画キャッシュ
- **禁止パターン**:
  - 4 項目以上に増やす（メイン枠サイズになる）
  - チェックボックスが描かれていない
  - 文字が長くサブ枠に収まらない
- **Remotion overlay 余白**: サブ枠（右側 30〜40%）に縦長で配置。字幕帯にかからないこと

## ranking_board

- **役割**: 「○選」「ランキング 3 位 / 2 位 / 1 位」「やってはいけない順」を見せる。リスト型台本の中盤〜終盤で使う主役。
- **会話パターン**: 「では 3 位から」 / 「1 位はこれ」
- **典型シーン**: ランキング型 / リスト型エピソードの章ごと、または最後のまとめ
- **構図ヒント**: ポディウム型（中央 1 位、左 2 位、右 3 位）または縦並びカード（上から 1 位）。順位ラベルは大きく、項目名はその下に短く。
- **配色**: 1 位 = 金、2 位 = 銀、3 位 = 銅。背景は薄グレー
- **ep902（パスワード）例**: ワーストパスワード TOP3「123456 / password / qwerty」
- **ep001（捨て候補）例**: 容量を食う TOP3「動画 / 写真 / アプリ内キャッシュ」
- **禁止パターン**:
  - 順位が読めない（ラベルが小さい / 同じ大きさ）
  - 項目名がメイン要素より大きい
  - 4 項目以上に増やす
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。ポディウムは中央 60% 帯に収める

## ui_mockup_safe

- **役割**: 抽象 UI（実在 UI 模写を避ける）。設定画面、入力フォーム、通知カードを「あるあるだが特定アプリではない」形で見せる。
- **会話パターン**: 「設定の中の」 / 「この画面が出たら」
- **典型シーン**: 操作系・セキュリティ系の説明シーン
- **構図ヒント**: 角丸カード＋抽象的な入力欄＋ぼかしロゴ＋ダミー文字（Lorem 風）。実在ブランド色・実在ロゴ形は避ける。
- **配色**: 白カード＋薄グレー枠＋青緑ボタン
- **ep902（パスワード）例**: 抽象的なログイン画面に「●●●●●●」の伏字
- **ep904（詐欺DM）例**: 偽ログイン画面風カード（青緑ではなくグレー寄りで「公式ではない」印象）
- **禁止パターン**:
  - 実在アプリのロゴ・カラーをそのまま再現する
  - 本物の URL や本物のサービス名を文字として書く
  - キャラと同じ画面に重ねて誰の画面か曖昧にする
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。UI カードは中央 50〜60% 帯に収める

## flowchart_scene

- **役割**: プロセス全体図を 1 枚で見せる。複数分岐のある判断フローや、長いプロセスの俯瞰。
- **会話パターン**: 「全体像はこう」 / 「分岐はここ」
- **典型シーン**: 中盤の俯瞰パート / 解決策の構造提示
- **構図ヒント**: 横向き or 縦向きフロー。ノードは角丸長方形、矢印は太め。分岐がある場合は菱形（判断ノード）。ノードは 4〜6 個に抑える。
- **配色**: ノード = 薄青緑、判断ノード = 薄黄、矢印 = 濃グレー
- **ep904（詐欺DM）例**: 「DM 受信 → 公式アプリ確認 → 公式から通知あり? → Yes 対応 / No 無視・通報」
- **ep001（写真整理）例**: 「容量不足検知 → 退避先選択 → 重複削除 → クラウド同期 → 完了」
- **禁止パターン**:
  - ノードが 7 個以上で読めない
  - 矢印が交差して順序がわからない
  - 文字情報のみでノード形が描かれていない
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。フロー全体は中央 60% 帯に収める

## contrast_card

- **役割**: 2 つの選択肢・概念を並列カードで比較する。before_after との違いは「時間軸ではなく性質の対比」であること。
- **会話パターン**: 「A タイプと B タイプ」 / 「無料版と有料版」 / 「A 派と B 派」
- **典型シーン**: 概念説明 / 中盤の選択肢提示
- **構図ヒント**: 左右 2 枚のカードを同サイズで並列。各カードにタイトル＋代表アイコン＋短い特徴 2〜3 行。中央に「VS」「OR」「比較」などの中継ラベル。
- **配色**: 左カード = 青系、右カード = オレンジ系。背景は薄グレー
- **ep902（2FA）例**: 「アプリ認証」 vs 「SMS 認証」
- **ep001（クラウド）例**: 「自動同期」 vs 「手動バックアップ」
- **禁止パターン**:
  - どちらが優位か曖昧で、視聴者が混乱する場面で使う（その場合は myth_vs_fact / before_after）
  - カードサイズが左右で違う
  - 中央のラベルがない
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。カードは中央 60% 帯に収める

## meme_like_diagram

- **役割**: ゆっくり系に合う少し笑える図解。比喩を 1 枚で押し出して理解を一気に進める。ただし安っぽいネットミーム画像にはしない。
- **会話パターン**: 強い比喩 / 「これって要するに〜」
- **典型シーン**: 中盤の比喩説明 / 印象づけが必要な場面
- **構図ヒント**: 比喩のオブジェクトをイラスト化（小銭の山 / ゴミ屋敷 / バケツリレー）。オブジェクトに短いラベルを付け、比喩対象を 1〜2 語で対応づける。
- **配色**: 中明度のパステル＋アクセント 1 色。彩度は控えめ
- **ep001（写真 1 枚削除）例**: 「スクショ 1 枚ずつ削除＝小銭集め」 → 小銭の山に「1MB」のラベル
- **ep001（アプリ内データ）例**: 「アプリ内データ＝ゴミ屋敷」 → 散乱した部屋
- **禁止パターン**:
  - ネット拾い物のミーム画像をそのまま使う
  - 文字だけで比喩を説明（イラストがない）
  - 比喩対象が不明（何を例えたものか伝わらない）
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。比喩オブジェクトは中央 50〜60% 帯に収める

## mini_story_scene

- **役割**: 短いストーリー仕立てで「ある人がこの罠にかかった」「こうやって解決した」を 3 コマ風に見せる。事件解説型と相性が良い。
- **会話パターン**: 「ある日〜」 / 「結局こうなった」
- **典型シーン**: 事件解説型の冒頭 / 体験談導入
- **構図ヒント**: 横 3 コマ漫画スタイル。各コマに小さな状況カット＋簡潔な状況ラベル。コマ間は薄い区切り線。
- **配色**: 各コマ同じ淡色で統一。最終コマだけアクセント色を入れて結末を強調
- **ep904（詐欺DM）例**: ①「DM が来た」 →②「リンクを踏んだ」 →③「アカウントが消えた」
- **ep001（容量不足）例**: ①「旅行先で撮影」 →②「容量不足で保存失敗」 →③「同じ瞬間は二度と来ない」
- **禁止パターン**:
  - 4 コマ以上にしてしまう（3 コマで完結させる）
  - コマ枠が描かれていない
  - 結末コマで「で、どうなった？」のまま終わる
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。3 コマ全体は中央 60% 帯に収める

## final_action_card

- **役割**: 視聴後 CTA。「今すぐやるべき 2〜3 アクション」を最後の 1 枚に集約する。最終行動ゲートの可視化。
- **会話パターン**: 「動画を閉じる前にこれだけやって」 / 「3 つだけ約束」
- **典型シーン**: 動画ラスト / s_final
- **構図ヒント**: 横 3 枚カード構成、矢印でつなぐ。各カードに番号＋短いアクション動詞＋小アイコン。最後のカードはひときわ強調（外周だけ濃色枠）。
- **配色**: 全カード青緑系で統一。最後のカードのみ濃色枠＋金色アクセント
- **ep904（詐欺DM）例**: ①公式アプリで確認 →②ブロック →③通報
- **ep001（写真整理）例**: ①重複削除 →②クラウド同期 →③テンプレ化
- **禁止パターン**:
  - アクションが「考える」「気をつける」など曖昧動詞
  - 4 アクション以上で長くなる（3 つに絞る）
  - 矢印がなくて順序がわからない
- **Remotion overlay 余白**: 上部左 36% タイトル空き、下部 20% 字幕。3 カードは中央 60% 帯に収める
