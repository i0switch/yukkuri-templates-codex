# 02_IMAGEGEN_PROMPT_PROMPT.md

ゆっくり解説 / ずんだもん解説動画の各シーン構図設計（image_directions YAML）を、
GPT-Image-2 が高品質に解釈できる日本語 `imagegen_prompt` 文字列へ変換するための LLM 向けプロンプト。

---

## 0. 役割定義

あなたは、image_direction を GPT-Image-2 が高品質に解釈できる日本語プロンプト文に変換する。
出力は **構図 → スタイル → 色 → アスペクト → 文字方針 → 禁止** の固定順とし、
必須キーワード（前景・中景・背景・下部20%・Remotion・禁止）を必ず本文に含める。

固定順を守らなかった、または必須キーワードのいずれかが欠落した出力は不合格として書き直す。

---

## 1. 入力フォーマット

LLM には次の3つを渡す：

- `image_directions`: `01_IMAGE_DIRECTION_PROMPT` で生成された YAML（各シーンの構図設計）
- `character_pair`: `RM`（霊夢・魔理沙）または `ZM`（ずんだもん・めたん）― 色味プリセット選択に使う
- `design_style_global`: 動画全体の image_style（例: "フラット・親しみ系" / "フラット・注意喚起広告風"）

入力 YAML には少なくとも次のキーが含まれる前提：

```yaml
image_directions:
  - scene_id: s01
    slot: main
    visual_type: hook_poster | danger_simulation | boke_visual | final_action_card | ...
    composition_type: smartphone_closeup | diagonal_flow | product_shot | three_cards | ...
    aspect: "16:9" | "1:1" | "9:16"
    foreground: 前景に置くもの（原文）
    midground: 中景に置くもの（原文）
    background: 背景に置くもの（原文）
    image_should_support: 台本のどの瞬間を支えるか（原文セリフ含む）
    color_palette: 配色指示
    image_text_examples: ["短語1", "短語2", ...]   # 画像内に入れてよい短語
    nlm_marker_id: "[FIG:N]"   # 任意・フォールバック用
```

---

## 2. 出力フォーマット（YAML、scene ごとにブロック）

scene × slot ごとに1ブロック。`imagegen_prompts` 配列でまとめる。

```yaml
imagegen_prompts:
  - scene_id: s01
    slot: main
    visual_type: hook_poster
    composition_type: smartphone_closeup
    aspect: "16:9"
    style: "フラット・注意喚起広告風"
    negative: "写真風、リアル調、文字、ロゴ、人物の顔、漫画的な吹き出しテキスト、長文日本語"
    imagegen_prompt: |
      ゆっくり解説 / ずんだもん解説動画のScene02メイン枠で使う高品質ビジュアル素材。
      この画像は、台本内の「（image_should_supportを引用）」を視覚的に補強する。
      visual_typeは「hook_poster」、composition_typeは「smartphone_closeup」。

      画面構図：
      前景には（foreground原文）。
      中景には（midground原文）。
      背景には（background原文）。
      視線は前景の指先からSTOPサインへ流れる。

      デザイン：
      日本の注意喚起広告とプロダクトデザインボードの中間のような完成度の高い画面。
      ただし実在ブランドや実在UIは模写しない。
      色は（color_palette）。
      光は柔らかい順光、影は薄め。
      画面下部20%は字幕とキャラ表示用に空ける。

      文字方針：
      画像内の文字は最大3語まで。
      正確な日本語タイトルや説明はRemotionで重ねるため、長文は入れない。
      入れてよい短語は（image_text_examples をカンマ区切り）。

      禁止：
      白背景に中央アイコンだけ、汎用素材、実在ロゴ、実在UI、既存キャラクター、写真風人物、長文テキスト、細かい表。
```

`imagegen_prompt` 内の `（...）` は入力 YAML の該当値で **必ず置換** する。
プレースホルダのまま出力したら不合格。

---

## 3. 必須記載事項（imagegen_prompt 本文に必ず含める語）

各シーンの `imagegen_prompt` 本文には以下のキーワードがすべて含まれている必要がある：

- 「前景」
- 「中景」
- 「背景」
- 「下部」「20」（下部20%余白）
- 「Remotion」（重ね用余白）
- 「禁止」

1つでも欠落していたら、その scene は不合格として書き直す。

---

## 4. プロンプト長制約

- 最低 **250 文字**
- 最大 **600 文字**

調整ルール：

- 短すぎる → 構図 / 色 / 光 / 視線誘導 を補強する
- 長すぎる → 禁止リストを `negative` フィールドに追い出して本文をスリムにする
- 文字数カウントは日本語文字 + 句読点 + 改行を含める

---

## 5. ジャンル別プリセット

`character_pair` と `design_style_global` から自動選択する。

### RM（霊夢・魔理沙、サイエンス問題提起・凶悪事件解説など）

- 配色: 青緑系・寒色寄り、危険部だけ赤
- スタイル: フラット研究室風 / フラット注意喚起広告風 / 解剖アイコン
- 雰囲気: 「実験データを見せている」「事件のメカニズムを示す」

### ZM（ずんだもん・めたん、雑学・親しみ）

- 配色: 暖色系・ピンクオレンジ・クリーム
- スタイル: フラット教室風 / 吹き出し / プロダクト広告風
- 雰囲気: 「説明イラスト」「広告ポスター内図解」

`design_style_global` が明示されている場合はそちらを優先する。
プリセットはあくまでベース。

---

## 6. 禁止表現（imagegen_prompt 本文に書いてはいけない語）

本文に登場した時点で不合格：

- 「白背景に中央アイコン」（単独表現として）
- 「シンプルなアイコン」（単独表現として）
- 「汎用」「どのシーンでも使える」
- 実在ブランド名: Twitter, X, Instagram, LINE, Apple, Google, Amazon, Meta, Facebook, TikTok など
- 実在キャラ名: 霊夢 / 魔理沙 / ずんだもん / めたん

ただし `negative` フィールドや「禁止」段落の **禁止対象として明示的に列挙する** 場合は許容する。
例: 「禁止：実在ロゴ、実在UI、既存キャラクター」は OK。
例: 「中央に Twitter のロゴ」は NG。

---

## 7. 文字方針セクションの固定文

各 `imagegen_prompt` の「文字方針」段落には以下を必ず書く（一字一句固定）:

- 「画像内の文字は最大3語まで。」
- 「正確な日本語タイトルや説明はRemotionで重ねるため、長文は入れない。」

この2文が欠けていたら不合格。

---

## 8. 禁止段落の固定文

各 `imagegen_prompt` の「禁止」段落には以下を必ず書く:

- 「白背景に中央アイコンだけ」
- 「汎用素材」
- 「実在ロゴ」
- 「実在UI」
- 「既存キャラクター」
- 「写真風人物」
- 「長文テキスト」

これらは「禁止：」の後にカンマ区切りで列挙するのが基本フォーマット。
シーンごとに追加禁止語（例: 「細かい表」「単なる鎖アイコン」）を足してよい。

---

## 9. 自己チェック

出力前に LLM 自身で以下を確認する。`[ ]` を `[x]` に書き換える形でチェックリストを通す：

- [ ] 全 `imagegen_prompt` が必須キーワード 6 種（前景・中景・背景・下部・20・Remotion・禁止）を含む
- [ ] 文字数 250〜600 の範囲内
- [ ] `negative` に最低限 `["写真風", "リアル調", "文字", "ロゴ", "人物の顔"]` が含まれる
- [ ] `aspect` が `"16:9"` / `"1:1"` / `"9:16"` のいずれか
- [ ] `visual_type` ごとの構図ヒントが反映されている（hook_poster なら冒頭フック感、final_action_card なら3ステップ構造、など）
- [ ] 入力 YAML のプレースホルダ `（...）` が全て実値に置換されている
- [ ] セクション順が「画面構図 → デザイン → 文字方針 → 禁止」固定順
- [ ] 文字方針の固定文2行が一字一句一致
- [ ] 禁止段落の固定7語がすべて含まれる

1項目でも未達なら、該当 scene を書き直してから出力する。

## 10. ep904 詐欺DM 用サンプル（s01 / s02 / s03 / s04 / s08）の完成例

LLM の few-shot 教師例。これらをそのまま参照しつつ、入力 YAML に合わせた本文を生成する。

### s01 サンプル

`visual_type: hook_poster, composition_type: smartphone_closeup`
`supports_moment: "当選DMに霊夢が食いつき、魔理沙が開くなと止める"`

```yaml
imagegen_prompt: |
  ゆっくり解説動画のScene02メイン枠で使う高品質な冒頭フック画像。
  台本の「当選DM来た、勝った？」「まず開くな、止まれだぜ」を視覚的に補強する。

  左側に大きなスマホのクローズアップ。
  画面には実在アプリではない抽象DM通知カード。
  通知内の短語は「当選」「確認」程度。
  右側には赤いSTOPサイン。
  スマホ通知へ伸びる指が、STOPサインの手前で止まっている。
  背景は薄いグレーの注意喚起広告ポスター風。
  上部左側にRemotionでタイトルを重ねられる余白を残す。
  下部20%は字幕とキャラ用に空ける。

  青緑、白、薄グレーを基調に、危険箇所だけ赤。
  高品質な日本の注意喚起広告とガジェット広告の中間のデザイン。
  実在ブランドロゴ、実在UI、既存キャラクター、写真風人物、長文テキストは禁止。
```

### s02 サンプル

`visual_type: danger_simulation, composition_type: diagonal_flow`
`supports_moment: "急かす文面に霊夢が焦り、魔理沙が焦らせ方だと指摘する"`

```yaml
imagegen_prompt: |
  ゆっくり解説動画のScene02メイン枠用の高品質ビジュアル。
  急かす詐欺DMが、焦った人にリンクを押させる心理を見せる。

  画面左上に抽象化された通知カード。
  通知内の短語は「24h」「停止」だけ。
  中央にカウントダウン時計。
  右下へ向かって赤い斜め矢印が伸び、押そうとする指を誘導している。
  指の手前に赤い停止ラインを置く。
  背景は淡いグレーのキャンペーンバナー風。
  下部20%は字幕とキャラ用に空ける。

  危険な赤と、冷静な青緑を対比させる。
  実在UI、ブランドロゴ、長文日本語、写真風人物は禁止。
```

### s03 サンプル

`visual_type: boke_visual, composition_type: product_shot`
`supports_moment: "無料ギフトに霊夢が釣られ、魔理沙がカード情報を狙うと訂正する"`

```yaml
imagegen_prompt: |
  ゆっくり解説動画のScene02メイン枠用。
  無料ギフトが実は釣り針になっていることを、少し笑えるが怖い商品広告風に表現する。

  中央に高級感のあるギフト箱。
  箱のリボンの奥から小さな釣り針が見えている。
  右奥に薄くカード情報を示す抽象カードがあり、赤い細い糸で狙われている。
  背景は淡いクリーム色の広告ポスター風。
  画像内テキストは「無料」だけなら可。
  下部20%は空ける。

  可愛いだけではなく、罠感が伝わる構図。
  実在ブランド、実在カード会社ロゴ、写真風人物、長文は禁止。
```

### s04 サンプル

`visual_type: danger_simulation, composition_type: diagonal_flow`
`supports_moment: "押すだけなら平気という誤解を、迷路に入る危険として見せる"`

```yaml
imagegen_prompt: |
  ゆっくり解説動画のScene02メイン枠用。
  DM内リンクを押すと、危険な入力画面へ誘導される流れを視覚化する。

  左上に怪しいリンク付き吹き出し。
  そこから赤い矢印が中央の迷路へ吸い込まれる。
  迷路の出口には、個人情報入力フォーム風の抽象カードがある。
  入力カードには赤い警告マーク。
  右側には薄く青緑の安全ルートも配置し、危険ルートとの対比を作る。
  背景は暗すぎないグレーのデザインボード風。
  下部20%は空ける。

  単なる鎖アイコンは禁止。
  実在UI、実在サービス名、長文テキスト、写真風人物は禁止。
```

### s08 サンプル

`visual_type: final_action_card, composition_type: three_cards`
`supports_moment: "止まる、公式確認、ブロックという最終行動を覚えさせる"`

```yaml
imagegen_prompt: |
  ゆっくり解説動画のScene02メイン枠用のラスト行動カード。
  視聴後にやることを3ステップで覚えさせる。

  横3枚のカード構成。
  左カードは赤いSTOPサイン。
  中央カードはスマホと盾の公式確認イメージ。
  右カードはブロックと通報を表す青緑アイコン。
  3枚を細い矢印でつなぐ。
  Webデザインボードのように整った余白と影を使う。
  画像内文字は「STOP」「確認」「ブロック」まで。
  正確な説明はRemotionで重ねる。
  下部20%は字幕とキャラ用に空ける。

  汎用アイコン集にせず、ラストCTAとして画面映えする完成度にする。
  実在ロゴ、実在UI、長文、既存キャラは禁止。
```
