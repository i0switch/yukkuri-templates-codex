# 素材ライセンス取り扱いメモ

Claude Code / Codex が動画を生成する際に**絶対に守らせる**ライセンス運用ルール。
本ファイルはプロンプトから毎回 Read される前提で書く。

**大前提**：ライセンスが明確でない素材は使わない。迷ったら止まってリュウドウに確認する。

---

## 0. 共通ルール

1. 全ての素材について、**取得元 URL・ライセンス名称・クレジット表記要否**を
   `script/{episode_id}/meta.json` に記録する。
2. クレジット表記が必要な素材を使った場合、動画 outro または概要欄に掲載する（AI は
   `script.yaml` の末尾に `credits` フィールドを追加して記録）。
3. 1 つのエピソード内で同一素材の重複使用は避ける。
4. 商用利用の可否はサイトごとに確認する。**「商用可」表記がないものは使わない**。

## 1. サイトごとの運用ルール

### 1.1 いらすとや（https://www.irasutoya.com/）

| 項目 | 内容 |
|---|---|
| 商用利用 | **1 素材あたり 20 点まで無料**で可（それ以上は有償ライセンスが必要） |
| クレジット表記 | 不要（任意） |
| 改変 | 可（加工・切り抜き OK） |
| 禁止 | 素材そのものの再配布、公序良俗に反する使い方、商品化（LINEスタンプ等） |
| 取得方法 | 記事ページ → 画像を右クリック保存 or curl でDL |

**運用ルール**：
- 1 動画 ＝ **1 素材 = 1 点**としてカウント。20 素材を超える動画を作る場合は注意喚起
- 複数エピソードにまたがる場合、どの動画でどの素材を使ったかを
  リポジトリ横断で集計できるよう `meta.json` の `source_url` で突合できる状態を保つ
- 画像 URL は `https://4.bp.blogspot.com/...` 等の CDN URL ではなく
  **記事ページ URL**（`https://www.irasutoya.com/YYYY/MM/blog-post_xxx.html`）を記録する

### 1.2 ニコニ・コモンズ（https://commons.nicovideo.jp/）

| 項目 | 内容 |
|---|---|
| 商用利用 | **素材ごとに異なる**。商用可マーク付きのみ使う |
| クレジット表記 | **素材ごとに異なる**。必要なものは outro/概要欄に記載 |
| 改変 | 素材ごとに異なる。明記のないものは避ける |
| 禁止 | 作者が明示的に許諾していない用途 |
| 取得方法 | 素材ページからDL |

**運用ルール**：
- 素材ページの「利用条件」セクションを必ず `meta.json` にそのまま転記
- 「ニコニコ動画内」「ニコニ・コモンズ対応サービス内」限定の素材は YouTube 投稿用の動画には使わない
- クレジット表記が必要なら、動画 outro のテロップと概要欄の両方に入れる

### 1.3 Pexels / Unsplash / Pixabay（フリー写真サイト）

| 項目 | 内容 |
|---|---|
| 商用利用 | 可（Pexels License / Unsplash License / Pixabay License） |
| クレジット表記 | 不要だが推奨 |
| 改変 | 可 |
| 禁止 | 被写体人物を侮辱する使い方、被写体の同一性を損なう加工での販売 |
| 取得方法 | 公式 API または直接DL |

**運用ルール**：
- 写真素材が必要な場合はここから。イラストは使わない（クオリティ差が目立つ）
- Unsplash は商用可だが被写体が特定人物の場合、パブリシティ権に注意
- 医療・薬理系の写真は被験者同意の取れていない写真がある可能性があるので、
  Pexels/Unsplash 由来でも **人物クローズアップは避ける**

### 1.4 DOVA-SYNDROME（https://dova-s.jp/）

| 項目 | 内容 |
|---|---|
| 商用利用 | 可（個人・法人問わず） |
| クレジット表記 | **原則不要**。ただし各曲の規約を個別に確認 |
| 改変 | 一部曲は不可。曲ページの規約を見る |
| 禁止 | 楽曲のみの再配布、音素材としての転売 |
| 取得方法 | 曲ページからMP3DL |

**運用ルール**：
- 曲ページ URL を `meta.json` に記録
- 曲ページに「**クレジット表記が必要**」の記載がある場合のみ outro/概要欄に表記
- DOVA-SYNDROME の曲でも作曲者によって規約が異なる → **毎曲必ず確認**

### 1.5 甘茶の音楽工房（https://amachamusic.chagasi.com/）

| 項目 | 内容 |
|---|---|
| 商用利用 | 可 |
| クレジット表記 | **任意（不要だが推奨）** |
| 改変 | 可 |
| 禁止 | 楽曲の再配布、アダルト系への使用 |

**運用ルール**：DOVA-SYNDROME と同じ扱い。

### 1.6 効果音ラボ（https://soundeffect-lab.info/）

| 項目 | 内容 |
|---|---|
| 商用利用 | 可 |
| クレジット表記 | 不要 |
| 改変 | 可 |
| 禁止 | 素材の直接再配布・転売 |

## 2. 音声合成エンジンのライセンス

### 2.1 VOICEVOX

| 項目 | 内容 |
|---|---|
| 商用利用 | 可 |
| クレジット表記 | **必須**。「VOICEVOX:ずんだもん」「VOICEVOX:四国めたん」など話者ごとに記載 |
| 表記場所 | 動画概要欄または outro テロップ |
| 規約 | https://voicevox.hiroshiba.jp/term/ |

**運用ルール**：
- 使用した話者全員を `meta.json` に記録
- outro シーンの `sub.items` にクレジットを入れるか、
  `script.yaml` の `credits` フィールドで管理し、概要欄生成時に反映

```yaml
# script.yaml の末尾に追加される
credits:
  voices:
    - "VOICEVOX:ずんだもん"
    - "VOICEVOX:四国めたん"
  bgm:
    - { title: "穏やかな日常", composer: "甘茶", site: "甘茶の音楽工房" }
  images:
    - { note: "一部素材：いらすとや" }
```

### 2.2 AquesTalk / SofTalk

| 項目 | 内容 |
|---|---|
| 個人・非商用 | 基本無料で利用可 |
| 商用利用 | **AquesTalk の商用ライセンスが別途必要になる場合がある**（AQUEST 社の規約確認） |
| クレジット表記 | 推奨 |
| 霊夢・魔理沙の声を「ゆっくり実況」として使う場合 | `AquesTalk License Free` の範囲内か要確認 |

**運用ルール**：
- YouTube 収益化する動画で RM ペアを使う場合は、リュウドウが AquesTalk Pro 等の
  商用ライセンスを保有しているか、事前に確認が取れている前提で進める
- 不明ならプロンプト実行を止めてリュウドウに確認する
- 概要欄に「ゆっくり音声素材：AquesTalk」と表記する

## 3. ImageGen 生成物の扱い（Codex 向け）

| 項目 | 内容 |
|---|---|
| 著作権 | 生成サービスの利用規約に従う |
| 商用利用 | 多くのサービスは商用可。ただし必ず利用規約を確認 |
| クレジット表記 | 通常不要（規約による） |
| 禁止 | 実在人物・既存キャラクター・ブランドロゴの生成と利用 |

**運用ルール**：
- 生成プロンプトに実在人物名・著作物名・ブランド名を入れない
- 生成画像に想定外の既存キャラ・ロゴが写っていないか目視確認
- `meta.json` に `imagegen_prompt` を必ず記録する

## 4. 動画 outro の定型クレジット欄

台本生成時に outro シーン（最後の scene）の `sub` に以下を自動挿入する：

```yaml
meta:
  layout_template: "Scene04"
scenes:
  - id: "s{N}"
    role: "outro"
    main:
      kind: "text"
      text: "ご視聴ありがとうございました"
    sub:
      kind: "bullets"
      items:
        - "音声：VOICEVOX（ずんだもん・四国めたん）"
        - "BGM：{曲名}（{サイト}）"
        - "素材：いらすとや 他"
```

RMペアなら `音声：AquesTalk（ゆっくり音声）` に差し替え。

---

## 5. 判定フローチャート

```
素材を使いたい
      ↓
商用利用OK？ → NO → 使わない・別素材探す
      ↓ YES
クレジット表記必要？ → YES → meta.json に記録 ＋ credits フィールドに追加
      ↓ NO
改変OK？ → NO → そのまま使う（加工しない）
      ↓ YES
meta.json に source_url と license を記録 → 使用OK
```

---

## 6. AI が迷ったら

以下の状況に陥ったら、**自律ループを止めてリュウドウに確認**する：

1. 素材ページに利用条件の記載が一切ない
2. 「〜のみ利用可」という限定条件があるが、動画投稿先が該当するか判断できない
3. AquesTalk の商用利用可否が不明
4. 生成画像に既存キャラやロゴが紛れ込んでいるか判断できない
5. 同一素材を 20 回以上使うエピソード（いらすとやルール抵触）

**判断に迷う = 止まる**。勝手に「まあ大丈夫だろう」で進めない。
