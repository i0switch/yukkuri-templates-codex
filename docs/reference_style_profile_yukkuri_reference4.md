# 参照4動画型 reference_style_profile

## 目的

この文書は、参照4動画の台本構造をそのままコピーするためではなく、ゆっくり解説として再利用できる「構成、テンポ、情報の出し方」を抽出するためのプロファイルである。

参照動画の台詞、固有表現、画像、サムネ文言は流用しない。抽出対象は、冒頭の入り方、章構成、疑問の置き方、証拠と反論の流れ、中盤の再フック、終盤の回収方法に限定する。

## 共通の再現要素

- 冒頭で「普通ではない謎、疑惑、違和感」を置き、視聴者の疑問を先に作る。
- 本編は `番号付きリスト` または `時系列深掘り` のどちらかに寄せる。
- 章ごとに小さな謎を提示し、事実、背景、仮説、未解明点の順に整理する。
- 霊夢、ずんだもん側は視聴者代表として疑問、驚き、言い換え、ツッコミを担当する。
- 魔理沙、めたん側は整理、背景説明、根拠、断定回避を担当する。
- 画像や資料は「1画面1情報」。長尺でも同じカードを引っ張りすぎない。
- 中盤に「ここまでの整理」と「まだ残っている一番大きな謎」を置く。
- 終盤は抽象感想ではなく、最初の謎に戻って回収する。

## reference_style 定義

### `listicle_mystery`

未解明の謎、都市伝説、歴史ミステリーなどを複数項目で紹介する型。

基本構成:

```text
OP -> 挨拶 -> 今回のテーマ -> 注意/前提 -> 1つ目の謎 -> 背景 -> 仮説 -> 未解明点 -> 次の謎 -> ... -> まとめ -> ED
```

必須条件:

- `reference_beat` に `numbered_case` を最低3回入れる。
- 各項目に `curiosity_gap` を置く。
- 各項目の最後に「まだ分かっていない点」か「次に気になる点」を置く。

### `scandal_case`

炎上、疑惑、噂、界隈の問題を扱う型。

基本構成:

```text
OP小芝居 -> テーマ提示 -> 注意書き -> ケース紹介 -> 疑惑点 -> 反論/限界 -> 現在の扱い -> 次ケース -> まとめ
```

必須条件:

- `reference_beat` に `disclaimer` を入れる。
- `evidence_role` は `claim` / `evidence` / `counterpoint` / `limit` を分ける。
- 断定できない内容は「噂」「指摘」「可能性」「断定はできない」と明示する。

### `crime_timeline`

事件、行方不明、人物像、経緯を時系列で追う型。

基本構成:

```text
OP -> 事件概要 -> 舞台/人物 -> 当日の流れ -> 分岐点 -> その後 -> 残る疑問 -> 注意喚起/風化防止 -> ED
```

必須条件:

- `reference_beat` に `timeline` を複数入れる。
- 被害者、関係者を茶化さない。
- 目的を注意喚起、再発防止、風化防止に置く。
- 事実、推測、報道ベースの情報を混ぜない。

### `deep_mystery`

人類、歴史、科学、進化などの大きな疑問を長尺で深掘りする型。

基本構成:

```text
OP -> 大きな疑問 -> 前提整理 -> 昔はどうだったか -> なぜ変わったか -> 他説/反論 -> 今に残る影響 -> 結末の問い -> ED
```

必須条件:

- 冒頭で大きな問いを1つに絞る。
- 中盤に「ここまで聞くとAに見えるが、本当に重要なのはB」を入れる。
- 終盤で最初の問いに戻り、視聴後に言える一言を作る。

## script.yaml に追加するメタ（任意・内部分析専用）

各シーンは、既存レンダーに影響しない任意フィールドとして次を持つことが**できる**。これらは構成分析と機械監査用の内部メタであり、`imagegen_prompt` 本文や Remotion 描画には**絶対に渡さない**。画像生成プロンプトに混入させない。

```yaml
reference_style: listicle_mystery
reference_beat: topic_setup
hook_type: unresolved_question
curiosity_gap: なぜ記録が残っているのに正体だけ分からないのか
evidence_role: background
next_reason: 次は似た伝承が現実に影響した例を見る
```

許可する `reference_beat`:

- `op_hook`
- `greeting`
- `disclaimer`
- `topic_setup`
- `numbered_case`
- `timeline`
- `background`
- `evidence`
- `counterpoint`
- `unresolved_point`
- `midpoint_rehook`
- `summary`
- `cta`

許可する `hook_type`:

- `unresolved_question`
- `strange_fact`
- `rumor_or_suspicion`
- `timeline_gap`
- `counterintuitive`
- `loss_or_warning`
- `none`

許可する `evidence_role`:

- `claim`
- `background`
- `evidence`
- `counterpoint`
- `limit`
- `interpretation`
- `summary`
- `none`

## 機械監査と別Codex監査の分担

機械監査で見るもの:

- 参照メタが80%以上のシーンに入っているか。
- 冒頭2シーン以内に強い `hook_type` があるか。
- `reference_beat` に中盤再フックと終盤回収があるか。
- `scandal_case` / `crime_timeline` で注意書きや断定回避があるか。
- 霊夢/ずんだもん側の疑問・反応が一定割合あるか。

別Codex監査で見るもの:

- 参照4本のような「続きが気になる流れ」になっているか。
- 謎や疑惑を煽るだけで、対処や整理が弱くなっていないか。
- 実在事件・実在人物を扱うときに敬意と断定回避が守られているか。
- 画像計画が「1画面1情報」になっているか。
