初心者向け 手動画像生成・持ち込み台本マニュアル

このマニュアルは、docs/pipeline_contract.md を正本ルールとした読みやすい運用版。 ユーザーは台本と画像だけ用意し、AI側が動画用ファイル、読み上げ、レンダリングまで進めるための手順をまとめる。

まず全体像

初心者はここだけ見れば流れがわかる。ユーザーがやることは「台本を渡す」「画像を生成して渡す」の2つだけ。

1. ユーザーが台本を渡す メモ、文章、箇条書きでもOK。AIが動画用に整える。
2. AIが画像依頼Markdownを出す どの画像を、どんな名前で作るかをsceneごとに指定する。
3. ユーザーが画像を返す AIが読み上げを作り、動画をレンダリングする。

ステップバイステップ

  1. ユーザーが台本を渡す 完成台本でも、ざっくりした原稿でも、箇条書きでもよい。AIは原文を source_manual_script.md に保存する。
  2. AIが台本を動画向けに加工する AIは話の順番、掛け合い、説明不足、山場を整え、planning.md、script_draft.md、script_final.md を作る。
  3. AIが画像生成依頼Markdownを出す AIは「この部分にふさわしい画像を生成してください」という形で、sceneごとの画像プロンプトと保存ファイル名を書く。出力先は image_prompt_v2.md。
  4. ユーザーが外部ツールで画像を生成する ChatGPT Images、Claude、Midjourneyなどで画像を作る。Codexを入れていなくてもここは問題ない。
  5. ユーザーが画像をAIに渡す ファイル名はAIが指定した通りにする。例: s01_main.png、s02_main.png。保存先は script/{episode_id}/assets/。
  6. AIが画像を受け入れる AIは画像の保存場所、ファイル名、生成ツール、権利確認を meta.json に記録する。ユーザーはJSONを直接触らなくてよい。
  7. AIが読み上げ音声を生成する script_final.md と script.yaml をもとに、ゆっくり / ずんだもんの読み上げ音声を作る。
  8. AIが動画をレンダリングする gateを通してからMP4を書き出し、最後に動画監査をする。完成条件は gate PASS、MP4あり、video audit PASS。

ユーザーに渡す画像生成依頼Markdownの例

AIは台本を加工したあと、こんなMarkdownを出す。 ユーザーは各プロンプトを画像生成ツールに貼って、指定されたファイル名で画像を返す。 

# 画像生成依頼

以下の画像を生成してください。
生成した画像は、指定ファイル名のままAIに渡してください。
画像サイズは 16:9。文字を入れる場合は、見出しだけを大きく入れてください。
会話全文は画像内に入れないでください。

## s01_main.png
保存先: script/{episode_id}/assets/s01_main.png
この部分にふさわしい画像を生成してください。

対象シーン:
月580円のサブスクを放置すると、1年で6,960円になる。
主人公が「少額だから平気」と思っていたが、合計額を見て焦る場面。

画像プロンプト:
家計簿アプリの画面に、小さな月額サブスクが積み上がって大きな年間支出になっている様子。
中央に「月580円が年6,960円に化ける」という大きな見出し。
不安感はあるが、暗すぎない。ゆっくり解説動画のメイン画像として見やすい構図。

## s02_main.png
保存先: script/{episode_id}/assets/s02_main.png
この部分にふさわしい画像を生成してください。

対象シーン:
使っていないアプリ、無料期間後の自動更新、家族共有の見落としをチェックする場面。

画像プロンプト:
スマホ画面に複数のサブスク項目が並び、危ない項目だけ赤い注意マークで強調されている。
中央に「見落としやすい3つの支払い」という大きな見出し。
チェックリスト風で、初心者にも一瞬で意味がわかる画像。

ユーザーがやること

  • 台本、メモ、箇条書きなどの元ネタを渡す。
  • AIから出た画像生成依頼Markdownを読む。
  • 外部画像生成ツールにプロンプトを貼る。
  • s01_main.png のように指定された名前で画像を保存する。
  • 生成した画像ファイルをAIに渡す。

AIがやること

  • ユーザー原文を source_manual_script.md に保存する。
  • 台本を動画向けに加工して script_final.md まで作る。
  • 画像生成依頼Markdownとして image_prompt_v2.md を出す。
  • 受け取った画像を assets/ に配置し、台帳を作る。
  • 読み上げ音声を生成し、動画をレンダリングする。

AI側が内部で残すファイル

ここから下はAIや運用者向け。初心者ユーザーは直接編集しなくてよい。 ただし、何が作られるかを確認したいときに見る。

audits/manual_intake.md 雛形

# manual_intake

mode: hybrid_user_script
source_script: source_manual_script.md
script_author: user
image_source: user_generated
rights_confirmed: true

notes:
- ユーザー原文を source_manual_script.md に保存済み
- AIが planning.md / script_draft.md / script_final.md へ整える
- 画像はユーザーが外部生成ツールで作成し、assets/sNN_main.png に保存する

meta.json.assets[] の画像エントリ例

{
  "file": "assets/s01_main.png",
  "source_type": "user_generated",
  "generation_tool": "ChatGPT Images",
  "rights_confirmed": true,
  "license": "user confirmed generated asset for this episode",
  "scene_id": "s01",
  "slot": "main",
  "purpose": "script_final直投げ型の挿入画像",
  "adoption_reason": "sceneの要点に一致",
  "imagegen_prompt": "image_prompt_v2.md と同じプロンプト"
}

確認コマンド

npm run gate:episode -- <episode_id>
npm run render:episode -- <episode_id> out/videos/<episode_id>.mp4
npm run audit:video -- <episode_id>

正式素材として扱えないもの

次の素材は、動画内に見た目として置けても、正式な画像生成済み素材としては扱わない。 gateや監査で止まる原因になる。

  • ローカルで作った仮PNG、SVG、HTML/Canvas/Pillowなどの自作画像
  • placeholder、fallback、local card、copied として作られた画像
  • 出所不明画像、権利確認がない画像
  • generation_tool が空の user_generated 画像
  • rights_confirmed: true がない画像
重要: Codex imagegen が未実行なら source_type: "imagegen" にはしない。 外部ツールでユーザーが作った画像は source_type: "user_generated" として受け入れる。

完了前チェックリスト

  • ユーザー原文が source_manual_script.md に保存されている。
  • 動画用に加工した完成台本 script_final.md がある。
  • 画像生成依頼Markdown image_prompt_v2.md がある。
  • ユーザーから返ってきた画像が assets/s01_main.png 形式で揃っている。
  • 画像は user_generated として受け入れ、generation_tool と rights_confirmed: true を記録している。
  • 読み上げ音声が生成されている。
  • npm run gate:episode -- <episode_id> がPASSしている。
  • npm run render:episode -- <episode_id> out/videos/<episode_id>.mp4 でMP4が出ている。
  • npm run audit:video -- <episode_id> がPASSしている。

失敗したときの見方

missing-manual-source-script

source_manual_script.md がない。持ち込み台本の場合は、ユーザー原文をこのファイルへ保存する。

missing-manual-intake または incomplete-manual-intake

audits/manual_intake.md がないか、必要項目が不足している。上の雛形を使って補う。

user_generated image assets require generation_tool

meta.json.assets[] の画像エントリに、使用した画像生成ツール名を入れる。

user_generated image assets require rights_confirmed: true

ユーザーが生成画像の利用権を確認したうえで、該当エントリに rights_confirmed: true を入れる。

image source_type must be imagegen or user_generated

画像の source_type が不正。Codex生成なら imagegen、外部ツールでユーザーが生成したなら user_generated にする。

このHTMLはユーザー操作用マニュアル。仕様や停止条件を変更するものではない。矛盾がある場合は docs/pipeline_contract.md を優先する。
