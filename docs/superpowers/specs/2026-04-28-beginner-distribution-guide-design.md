# Beginner Distribution Guide Design

## 目的

配布先の完全初心者が、ZIPで受け取った `yukkuri-templates` を迷わず初回セットアップし、Claude Code へ依頼して、ゆっくり解説 / ずんだもん解説動画を生成できるようにする。

生成先は次の1ファイルにする。

```text
C:/Users/i0swi/Desktop/ナレッジ/ryudos-product-hub/public/yukkuri-zundamon-generator/guide/index.html
```

## 読者前提

- 完全初心者向けに書く。
- Windows を主対象にする。
- Mac は軽い注意書きだけにする。
- 配布は ZIP 前提にする。
- Git は不要として扱う。
- Claudeアプリ、Claude Code、Codexアプリ版は導入済み前提にする。
- 未導入の場合の確認ポイントは薄く置く。

## ガイド全体構成

`index.html` は1ページ完結の初心者オンボーディング型にする。既存のテンプレート画像一覧は削除せず、ページ後半の「テンプレート一覧」として包括する。

上から読む順番は次にする。

1. このアプリでできること
2. 配布ZIPの展開場所
3. 事前に入れておくもの
4. 初回セットアップ
5. 通常の動画生成の流れ
6. AIに投げるプロンプト例
7. 自作台本を使う方法
8. Codexなしで自前画像を使う方法
9. トラブル時の見方
10. 既存テンプレート一覧

## セットアップ説明

説明は専門用語から入らず、行動順で書く。

- ZIPを展開する。
- PowerShellでフォルダを開く。
- `npm install` を1回だけ実行する。
- VOICEVOXを使う場合は、VOICEVOXを起動してから生成する。
- ゆっくり音声系は、インストール先とライセンス注意だけ書く。
- 完成動画は `out/videos/{episode_id}.mp4` に出ると説明する。

音声まわりは、どちらも「基本はインストールするだけで設定不要」という前提で扱う。ただし、VOICEVOXとAquesTalk系のライセンス確認は目立つ注意ボックスにする。

## プロンプト例

ユーザー提供の例をベースに、コピペしやすい形で載せる。

```text
2分程度の動画を生成してください。
ゆっくりで生成してください。
使用したいテンプレはscene-12_classroom-bubbles.md
使用するテンプレートは必ず厳守してください。

ジャンル
AI副業で失敗する人の共通点

ツール集めで満足、商品設計なし、集客導線なし。
フック：「AIを使えば稼げる、と思った人から脱落します」

既存の台本は流用しないでください。
image genスキルを使用して画像の生成を実施してください。
動画書き出しはHDでお願いします。

完成まで自動で決定して進めて。
```

その下に、変更する場所を初心者向けに説明する。

- 動画の長さ
- ゆっくり / ずんだもん
- テンプレート名
- ジャンル
- フック
- 画像生成をCodexでやるか、自分で用意するか
- HD / FullHD などの書き出し解像度

## 自作台本フロー

内部名の `hybrid_user_script` は必要な場所だけに留め、見出しは「自分の台本をそのまま使いたい場合」にする。

説明する内容は次。

- ユーザー原文を渡せる。
- AIが動画用に整形する。
- `source_manual_script.md` に原文を残す仕組みがある。
- `planning.md`、`script_draft.md`、`script_final.md` を作る。
- `script_final.md` のレビュー後に YAML / 画像 / 音声 / レンダーへ進む。
- 自分の台本でも、完成判定は gate、render、video audit まで必要。

## 自前画像フロー

「Codexがない人」「画像生成ツールを自分で使いたい人」向けの章にする。

説明する内容は次。

- AIに `image_prompt_v2.md` を作らせる。
- 読者が任意の画像生成ツールで画像を作る。
- 画像を `script/{episode_id}/assets/sNN_main.png` に保存する。
- `meta.json` では `source_type: "user_generated"` として扱う。
- `generation_tool`、`rights_confirmed: true`、`license` が必要。
- placeholder、fallback、local card、出所不明画像は使わない。

## テンプレート一覧

既存の `scene-01-rm-guide.png` から `scene-21-rm-guide.png` までを再利用する。

カードには次を表示する。

- Scene番号
- templateファイル名
- プレビュー画像
- クリックで画像を別タブ表示

今回の実装では画像の再生成はしない。

## 見た目

- 上部に大きいタイトルと「最短でやること」カードを置く。
- 目次ボタンで各章へジャンプできるようにする。
- 各章は白いカードで区切る。
- コマンドやプロンプトは濃い背景のコピペ用ボックスにする。
- 注意事項は黄色系、ライセンスや権利まわりは赤系のボックスにする。
- テンプレ一覧は既存画像を使ったカードグリッドにする。
- スマホでも読めるレスポンシブCSSにする。

## 確認方法

実装後に次を確認する。

- `index.html` に主要セクションが入っている。
- プロンプト例がコピペしやすい。
- `scene-01-rm-guide.png` から `scene-21-rm-guide.png` まで参照している。
- ZIP配布、Windows主対象、Mac軽い注意書き、Git不要の説明になっている。
- Codexなし自前画像フローと自作台本フローが含まれている。
- HTMLとして静的に成立している。

## 非スコープ

- テンプレート画像の再生成はしない。
- 複数HTMLへの分割はしない。
- GitHub clone 前提の導入手順にはしない。
- 音声ツールの細かい個別設定チュートリアルは作らない。
- この作業ではコミットを作らない。コミットはユーザーが明示した場合だけ行う。
