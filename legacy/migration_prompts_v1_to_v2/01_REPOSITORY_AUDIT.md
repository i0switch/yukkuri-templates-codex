# 01_REPOSITORY_AUDIT: 既存リポジトリ調査

## 目的

既存リポジトリ内の、台本品質低下・画像品質低下・監査すり抜けの原因を洗い出す。
まだ大規模編集はしない。

## やること

以下のファイル・ディレクトリを確認してください。

- `CLAUDE.md`
- `AGENTS.md`
- `README.md`
- `_reference/script_prompt_pack/`
- `prompts/`
- `scripts/`
- `script/` または `scripts/` 配下のエピソード生成物
- 画像生成関連ファイル
- audit関連ファイル
- template関連ファイル

## 探すべき旧ルール

以下を含む記述を洗い出してください。

- 初稿段階から表示都合で短くする指示
- `script.yaml` を直接生成する指示
- `scene_format`, `hook_type`, `visual_type`, `boke_or_reaction`, `myth_vs_fact` などの穴埋め型テンプレ
- 霊夢/ずんだもんが質問役だけになる構造
- 魔理沙/めたんが訂正役だけになる構造
- 画像生成プロンプトに抽象メタタグを渡す構造
- 画像内に日本語長文、表、矢印、チェックリスト、金額を描かせる指示
- ファイル存在確認だけで画像監査PASSにする処理
- gate通過を品質PASS扱いする処理

## 出力するもの

`docs/repository_audit_v2.md` を作成してください。

内容:

```md
# Repository Audit v2

## 調査対象

## 発見した旧ルール

| ファイル | 問題箇所 | なぜ問題か | 対応方針 |
|---|---|---|---|

## 削除または無効化すべき記述

## 残すべき既存仕様

## 新方針と衝突する箇所

## 次フェーズで編集すべきファイル
```

## 注意

- このフェーズでは原則として大きな編集をしない。
- ただし `docs/` が存在しない場合は作成してよい。
- 不明点は推測で埋めず、`UNKNOWN` と書く。
