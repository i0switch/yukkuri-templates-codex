# 台本生成プラン (ep001-yukkuri-cognitive-illusions)

## 1. 入力整理
- テーマ：人間の脳がバグる、日常の認知の錯覚7選
- 想定尺：約5分（300秒前後）
- キャラペア：RM（霊夢・魔理沙）
- 使用テンプレート：Scene04（whiteboard-desk）
- 仮定：日常で誰もが体験する認知バイアスを「自分の脳がバグってる」という切り口で7つ紹介。具体例ベース・カジュアル雑談。

## 2. テンプレート読取（Scene04）
- main_content：白板下半分（1720x680）。シンプルな図解・アイコン・短いキーワードを置く中心エリア。
- sub_content：なし（メイン1枚で勝負）
- subtitle_area：下部グレー机帯（透明バー型、白文字）。`dialogue[].text` をここに表示。
- title_area：上部（1720x100）。各章タイトル（フック型）を表示。
- character_layout：左下（霊夢、x=180）／右下（魔理沙、x=1740）、机の上に立つ
- avoid_area：上部タイトル帯と下部字幕帯の真ん中（高さ約180-860px）が main 描画域なので、構図はほぼ画面中央。タイトル・字幕・キャラに被らないこと。

## 3. 企画角度
- 採用角度：誤解訂正型（自分の判断は意外と当てにならない、を笑いで気づかせる）
- 視聴者が最初に持つ疑問：「自分の脳もバグってるの？」
- 一般的な誤解：「私は冷静に判断してる」
- 意外な結論：脳のバグは欠陥ではなく、進化の節約設計
- 視聴後に言える一言：「人間って7つもバグ抱えて生きてる」

## 4. 掛け合い方針
- 視聴者代表キャラの立ち位置：霊夢が「自分は当てはまらない」と油断 → 例で次々ハマって焦る
- 解説役の立ち位置：魔理沙が冷静に名前と仕組みを置く、深追いしすぎない
- 主な勘違い：「私はちゃんと考えて選んでる」「気のせい」「思い込みって弱い人がするもの」
- 感情の流れ：油断 → 驚き → 焦り → 笑い → 受容 → 行動
- 口癖の偏り対策：「だぜ」を3連続で続けない。霊夢の「え？」「マジで？」「うわぁ…」「やばい」を分散

## 5. ボケパターン3案
1. 霊夢が極端な解決策を出す（例：「じゃあ脳を信じない！」）
2. 霊夢が雑なたとえを出す（例：「机の上のレシートみたいな？」）
3. 霊夢が当てはまっていることに気づいて自爆する（例：「私さっき魔理沙の服褒めたばっかなのに…」）

## 6. 冒頭フック案
1. 損失型：「今日選んだ服、実は脳にダマされて選ばされてるかもしれないぜ」
2. 数字型：「人間の判断、その7割は錯覚で動いてる」
3. 誤解訂正型：「自分は冷静、って思った時点で1個目のバグ発動だぜ」

採用：3（誤解訂正型）を主軸に、数字を補強する形

## 7. 尺・密度設計
- 想定尺：5分前後（300秒）
- 目標シーン数：10シーン（s01-s10）
- 目標本編セリフ数：100セリフ前後（各シーン10±2）
- 1シーン平均セリフ数：10
- 中盤再フック位置：s05（5/10 = 50%地点）
- ずんだもん語尾目標：対象外（RM）
- 魔理沙「だぜ」目標：30〜50%
