# ep913-zm-notification-overload 台本プラン（Script Prompt Pack 監査済み）

## 1. 入力整理

- テーマ：スマホ通知管理（通知全部ONが集中を破壊する）
- 想定尺：3分（180秒）
- キャラペア：ZM（ずんだもん・四国めたん）
- 使用テンプレート：Scene16（SF3段パネル暗）
- 仮定：視聴者は通知を全部ONにしていて問題と思っていない

## 2. テンプレート読取

- main_content：中央メインパネル（x:70 y:130 w:1780 h:720）
- sub_content：なし
- subtitle_area：下部overlay（x:70 y:880 w:1780 h:170）

- character_layout：両端ギリギリ配置
- avoid_area：下部20%はキャラ・字幕用に空ける

## 3. 企画角度

- 採用角度：「数字で刺す」型
- 視聴者が最初に持つ疑問：通知を全部確認して何が悪いのか
- 一般的な誤解：通知は素早い対応のためにすべき
- 意外な結論：通知1回で23分の集中が消えている
- 視聴後に言える一言：SNS通知をOFFにしてチェック時間を決めるだけで2時間戻る

## 4. 掛け合い方針

- ずんだもん：浅い理解、驚き、ボケ、言い換えを担当（視聴者代表）
- 四国めたん：数字と手順で刺し、対処まで落とす（解説役）
- 主な勘違い：通知は全部見ないといけない
- 感情の流れ：驚き → ショック → 「でも怖い」 → 納得 → 実行決意
- ずんだもん語尾目標：「のだ」「なのだ」を全セリフの20〜40%

## 5. ボケパターン3案

1. 「通知は全部見ないといけないのだ！」（全部ON信仰）
2. 「いいねの通知が楽しいのだ！」（SNS依存）
3. 「全部OFFにしたら大事なの見逃すのだ」（怖い系誤解）

## 6. 冒頭フック案

1. 「通知を全部ONにしてる人が毎日失う時間の話」（損失型）
2. 「通知1回で23分消える話」（数字型）
3. 「スマホに1日2時間盗まれてる人の話」（泥棒比喩型）

## 7. 尺・密度設計

- 想定尺：180秒（3分）
- 目標シーン数：8
- 目標本編セリフ数：80
- 1シーン平均セリフ数：10
- 中盤再フック位置：s05（通知OFF後28%向上の数字フック）
- ずんだもん「のだ」目標：全セリフの20〜40%
- meta.image_style：太線フラット図解。SF黒背景でも読める高コントラスト。緑・シアン・黄アクセント

## 9. final_action（最終行動）

- SNS通知を設定から確認してOFFにする
- 明日の仕事時間中に通知チェックを1時間に1回だけに試す
- 集中タイムを記録して効果を見る

（2アクション以上：確認・試す・チェック・記録 を含む）
