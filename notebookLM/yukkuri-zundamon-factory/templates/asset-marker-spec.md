# 素材プレースホルダー仕様

台本内に埋め込む視覚素材マーカーの定義。
Claude Code はこの仕様に従って台本から素材要求を抽出し、
NotebookLM で生成・取得・マッピングする。

---

## 📌 マーカー書式

```
[<TYPE>:<n>] <要求内容1行説明>
```

- **必ず行頭に配置**（前後に話者発言を置かない）
- `<n>` は各タイプごとに 1 から連番
- 説明文は 40 字以内、名詞句中心、「〜の図」等の冗長表現禁止

---

## 🎨 タイプ一覧

| TYPE | 対応する NotebookLM Studio Content | 用途 |
|------|-------------------------------------|------|
| `FIG` | `infographic`（横長 / プロフェッショナル） | タイトルカード・全体像の単枚インフォ |
| `INFO` | `infographic`（縦長 / 情報密度高） | セクション内の詳細情報グラフィック |
| `MAP` | `mind_map` | 概念の関係性・分類図 |
| `SLIDE` | `slides` | まとめ・比較表・ポイント列挙 |
| `VIDEO` | `video`（オプション） | 30秒程度の補足映像（使用頻度低） |

---

## ✅ 配置ルール

1. **導入**：`[FIG:1]` 必須（タイトル/全体像）
2. **各本編セクション**：最低 1 つの視覚素材
   （`[INFO:n]` または `[MAP:n]`）
3. **まとめ**：`[SLIDE:1]` 必須（3点サマリー推奨）
4. **同一発話内への複数配置禁止**（テンポを崩すため）
5. **連続配置禁止**（マーカー間に最低 1 発話を挟む）

---

## 🧪 抽出正規表現（Claude Code で使用）

```regex
^\[(FIG|INFO|MAP|SLIDE|VIDEO):(\d+)\]\s+(.+)$
```

キャプチャ：
- `$1` = TYPE
- `$2` = 番号
- `$3` = 要求内容

---

## 📦 生成物マッピング規則

台本マーカー → 保存パス → 最終台本での置換形式

```
[FIG:1]  → output/assets/<slug>/fig_1.png    → ![fig_1](../assets/<slug>/fig_1.png)
[INFO:1] → output/assets/<slug>/info_1.png   → ![info_1](../assets/<slug>/info_1.png)
[MAP:1]  → output/assets/<slug>/map_1.png    → ![map_1](../assets/<slug>/map_1.png)
[SLIDE:1]→ output/assets/<slug>/slide_1.pdf  → [slide_1.pdf](../assets/<slug>/slide_1.pdf)
[VIDEO:1]→ output/assets/<slug>/video_1.mp4  → [video_1.mp4](../assets/<slug>/video_1.mp4)
```

---

## 🚨 監査対象項目

- [ ] タイトル直後に `[FIG:1]` がある
- [ ] まとめに `[SLIDE:1]` がある
- [ ] 各セクションに最低 1 つのマーカー
- [ ] 番号が連番になっており欠番・重複なし
- [ ] 全マーカーに対応する生成物ファイルが存在する
- [ ] 同一発話内に複数マーカーが無い
- [ ] マーカーの説明文が 40 字以内
