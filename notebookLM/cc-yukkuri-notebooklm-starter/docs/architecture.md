# アーキテクチャ概要

## 目的
テーマを渡すと、Claude Code が以下を順番に回せる状態を作る。

1. 台本生成
2. シーン分解
3. 素材要件定義
4. NotebookLM 素材生成
5. 素材監査
6. 差し戻し / 再生成

## 層

### 1. 人間入力
- テーマ
- ターゲット
- 動画尺
- トーン
- 参考資料
- 必須で触れる論点

### 2. 台本層
- ゆっくり台本
- ずんだもん＆めたん台本

### 3. 素材要件層
- Scene ごとの素材目的
- 挿入位置
- 必須要素
- NG要素
- NotebookLM に渡す要約プロンプト

### 4. NotebookLM 層
- Notebook 作成
- 台本 / 補助資料投入
- infographic / slide_deck / report / video 生成
- ダウンロード

### 5. 監査層
- 適合性評価
- ズレの説明
- 再生成指示の作成

## 重要設計

### A. 台本と素材要求は分ける
台本そのものと、素材に必要な要件は別管理にする。  
理由: 台本を修正しても、素材要求の比較がしやすくなるため。

### B. シーン番号を固定する
`Scene 01`, `Scene 02` のように固定し、後段の台帳や監査が同じIDを参照できるようにする。

### C. NotebookLM の成果物をそのまま「採用済み」にしない
NotebookLM は概念図・説明図・スライド候補づくりに向いている。  
一方で動画本編にそのまま入れるとズレることもあるため、監査層を必須にする。

### D. ずんだもん＆めたん版と、ゆっくり版は別最適化
同じテーマでも会話テンポやボケ / ツッコミ密度が違うため、台本テンプレを分ける。
