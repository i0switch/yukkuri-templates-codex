# Codex 監査依頼書

Claude Code が 44 枚を生成した後、この文書をそのまま Codex に渡す。

---

## 依頼内容

以下のプロジェクトで生成された静止画 44 枚（22 シーン × RM/ZM 両ペア）を監査してください。

### プロジェクト概要
- ツール: Remotion（React ベースの動画生成フレームワーク）
- 目的: ゆっくり解説動画の**静止画テンプレート**。将来的に動画化される前提
- 背景: 22 種類の画像（`public/backgrounds/bg-01.jpeg` ... `bg-22.jpeg`）をそのまま使用
- キャラ: 4 種（reimu/marisa/zundamon/metan）の顔アイコン
- ペア: RM = 霊夢＋魔理沙、ZM = ずんだもん＋めたん
- 各シーンに 4 種類のエリア（メイン/サブ/タイトル/字幕）があり、それぞれに
  プレースホルダラベル（「ここはメインコンテンツエリア」等）が表示される

### 監査対象ファイル
- `out/scene-01-rm.png` ... `out/scene-22-rm.png` （22 枚）
- `out/scene-01-zm.png` ... `out/scene-22-zm.png` （22 枚）
- `docs/all-scenes-rm-grid.png`（RM 版一覧）
- `docs/all-scenes-zm-grid.png`（ZM 版一覧）

### チェック項目

各画像について、以下 6 軸で判定してください：

#### A. キャラ配置の妥当性
- [ ] キャラが画面外に出ていない
- [ ] キャラが背景の重要装飾（イラスト部分、枠、機材、ホワイトボード枠など）を覆っていない
- [ ] キャラの左右配置が自然（字幕帯の左右端に被る / 机の上 / 並んで立つ 等）
- [ ] zundamon / metan で顔ではなく足先だけ映っている事故がない
- [ ] RM と ZM で同じシーンなら配置が揃っている（キャラ種類以外の要素は同一）

#### B. エリアラベルの可視性
- [ ] メインコンテンツエリアのラベル「ここはメインコンテンツエリア」が見える
- [ ] サブコンテンツエリア（該当シーンのみ）のラベルが見える
- [ ] タイトルエリア（該当シーンのみ）のラベルが見える
- [ ] 字幕エリアのラベル「ここは字幕エリア」が見える
- [ ] ラベルが背景と同化して読めなくなっていない（背景が暗いなら明るいテーマ、明るいなら暗いテーマで描画されているか）
- [ ] ラベルが他の要素で覆われていない

#### C. エリア座標の妥当性
- [ ] メインコンテンツエリアが背景の「想定された余白」に収まっている
- [ ] サブコンテンツエリアが適切な位置にある
- [ ] 字幕エリアが背景の字幕用スペース（下部白帯/黒帯/吹き出し等）と一致している
- [ ] エリア同士が過剰に重なっていない

#### D. 背景画像の扱い
- [ ] 背景画像がそのまま（加工なし）で使われている
- [ ] 背景画像が `objectFit: cover` で適切にフィットしている
- [ ] 背景が歪んでいない・切れすぎていない

#### E. RM/ZM 対応
- [ ] 同じシーンの RM 版と ZM 版で、キャラ以外の要素（背景・エリア・字幕帯の形）が完全に一致
- [ ] 両ペアとも同じクオリティ

#### F. 総合評価
- [ ] 動画化して字幕・コンテンツを差し替えたとき、違和感なく使えそうか

### 評価スケール

| 記号 | 意味 |
|---|---|
| ✅ | 問題なし |
| 🟡 | 軽微な改善余地あり（プロダクション許容範囲内） |
| 🔴 | 修正が必要 |
| ⚫ | 致命的問題（エリア崩壊・キャラ配置ミス・ラベル不可視） |

### 合格基準
各画像で 🔴 と ⚫ が 0 件、🟡 は 3 件以下。

### 出力フォーマット

各画像について Markdown で以下の形式で報告してください：

```markdown
## scene-XX-rm.png

### A. キャラ配置
- 左キャラ位置 ✅
- 右キャラ位置 🟡 （背景の机の縁に顔が少し被っている）
- ...

### B. ラベル可視性
- メインコンテンツラベル ✅
- 字幕ラベル ✅
- ...

### C. エリア座標 / D. 背景 / E. RM/ZM / F. 総合
（同様）

### 集計
- ✅: 18 / 🟡: 2 / 🔴: 0 / ⚫: 0
- 判定: 合格

### 修正推奨（🔴/⚫ がある場合のみ）
- なし
```

### 特に重点的に見てほしい箇所

1. **画像 14** は 5 エリア構成で一番複雑。左右下部のガラス枠にキャラがちゃんと収まっているか。
2. **画像 2, 3, 10, 13** は 3 エリア構成。右サブ枠にラベルが入っているか。
3. **画像 12, 19** は吹き出し字幕。吹き出しの尻尾の向きが自然か。
4. **画像 8** のタイトル札、**画像 15** の左上タイトル枠。装飾をキャラが覆っていないか。
5. **画像 4, 20** はホワイトボード構造が近似。机の上にキャラがちゃんと乗っているか。
6. **暗い背景（3, 6, 13, 15, 16）** でエリアラベルが見えるか。

### 修正指示の返却方法

```markdown

- 修正方法: [LAYOUT の定数をこう変更する等]

### scene-YY-zm
- 問題: ...
- 修正方法: ...
```
